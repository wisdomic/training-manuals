9.1 แชร์ Notebook กับเพื่อนร่วมงาน

NotebookLM สามารถแชร์ Notebook ให้ผู้อื่นเข้าถึงได้ เหมือนกับ Google Docs ทำให้ทำงานร่วมกันเป็นทีมได้สะดวก

วิธีแชร์ Notebook

1
เปิด Notebook ที่ต้องการแชร์

เข้า Notebook → คลิกปุ่ม Share (มุมบนขวา)

2
เพิ่ม Email ผู้ร่วมงาน

พิมพ์ Email (Gmail) ของเพื่อนร่วมงาน → เลือกสิทธิ์ (Viewer / Editor)

3
กดส่งคำเชิญ

คลิก Share → ผู้รับจะได้รับ Email แจ้งเตือน

ระดับสิทธิ์การเข้าถึง

ระดับ สิทธิ์ที่ได้ เหมาะสำหรับ
Viewer ดู Sources, ถาม Chat, ดู Notes ได้ แต่แก้ไขไม่ได้ ผู้บริหาร, เจ้าหน้าที่ที่ต้องการค้นข้อมูล
Editor ทำทุกอย่างได้ — เพิ่ม/ลบ Sources, แก้ Notes, สร้าง Audio ทีมงานที่ดูแล Notebook ร่วมกัน

ภาพ: หน้าจอ Share Notebook + กำหนดสิทธิ์

💡 Tips: การจัดการทีม
  • กำหนดคน 1-2 คน เป็น Editor ดูแลเนื้อหา (Admin ของ Notebook)
  • คนอื่นๆ เป็น Viewer — ใช้งานถามตอบได้ แต่ไม่แก้ไข Sources
  • สร้าง Notebook แยกตามทีม/หน่วยงานย่อย
  • ตั้งชื่อ Notebook ให้ชัดเจน เช่น "[ฝ่ายบริการ] คู่มือสิทธิ์คนพิการ"

9.2 สร้าง Knowledge Base องค์กร

Knowledge Base (ฐานความรู้) คือการรวบรวมเอกสาร ระเบียบ คู่มือ ขององค์กรไว้ในที่เดียว ที่ทุกคนค้นหาได้ด้วย AI

โครงสร้าง Knowledge Base แนะนำ (5 Notebooks)

📋 NB1: กฎหมาย/ระเบียบ
  • พ.ร.บ. ส่งเสริมฯ คนพิการ
  • ระเบียบ พก. ทุกฉบับ
  • ประกาศกระทรวง
  • หนังสือเวียน
📖 NB2: คู่มือปฏิบัติงาน
  • คู่มือขึ้นทะเบียนคนพิการ
  • คู่มือการให้บริการ
  • ขั้นตอนการเบิกจ่าย
  • แบบฟอร์มต่างๆ
📊 NB3: รายงาน/สถิติ
  • รายงานประจำปี
  • สถิติคนพิการในพื้นที่
  • ผลงานเด่น
  • แผนยุทธศาสตร์
🎓 NB4: อบรม/พัฒนาบุคลากร
  • เอกสารอบรมทั้งหมด
  • คู่มือใช้งานระบบ
  • FAQ พนักงานใหม่
  • วิดีโอสอนงาน (YouTube links)
📢 NB5: สื่อ/ประชาสัมพันธ์
  • เนื้อหาสื่อ PR ทั้งหมด
  • ข่าวประชาสัมพันธ์
  • แผ่นพับ/โบรชัวร์
  • บทความเผยแพร่

วิธีจัดหมวดหมู่ Sources

หลักการ: แบ่ง Notebook ตาม "หัวข้อที่คนถามบ่อย" ไม่ใช่ตาม "ประเภทเอกสาร"

  • ✅ ดี: "สิทธิ์คนพิการ" (รวมทุกเอกสารเกี่ยวกับสิทธิ์)
  • ❌ ไม่ดี: "PDF ทั้งหมด" (รวมทุก PDF ไม่ว่าเรื่องอะไร)
  • ✅ ดี: "การจ้างงานคนพิการ" (รวม พ.ร.บ. + ระเบียบ + คู่มือ เฉพาะเรื่องจ้างงาน)
  • ❌ ไม่ดี: "พ.ร.บ. ทุกฉบับ" (รวมกฎหมายทุกเรื่องปนกัน)
💡 Tips: จำนวน Sources ที่เหมาะสม
  • แต่ละ Notebook ควรมี 5-30 Sources (ไม่เกิน 50)
  • Sources ควรเป็นเรื่องเดียวกันหรือเกี่ยวข้องกัน
  • ถ้ามีเอกสาร 100+ ฉบับ ให้แบ่งเป็น 3-5 Notebooks
  • ตั้งชื่อ Notebook ให้คนอื่นเห็นแล้วเข้าใจทันที

9.3 ออกแบบระบบตอบคำถามภายในองค์กร

ใช้ NotebookLM เป็น "ระบบตอบคำถาม" ให้เจ้าหน้าที่สอบถามข้อมูลได้ทันที โดยไม่ต้องค้นเอกสารเอง

แนวคิด: NotebookLM เป็น Q&A Bot ขององค์กร

❓

เจ้าหน้าที่มีคำถาม

"เบี้ยคนพิการจ่ายวันไหน?"

🤖

ถาม NotebookLM

เปิด Notebook ที่เกี่ยวข้อง → พิมพ์คำถาม

✅

ได้คำตอบ + Citation

คำตอบพร้อมอ้างอิงจากเอกสารจริง

ขั้นตอนออกแบบระบบ

1
รวบรวมคำถามที่ถูกถามบ่อย

สอบถามเจ้าหน้าที่ว่าคำถามที่ถูกถามบ่อยคืออะไร → จัดหมวดหมู่

2
รวบรวมเอกสารที่ใช้ตอบ

หาเอกสาร/ระเบียบที่ใช้ตอบคำถามเหล่านั้น → อัปโหลดเข้า Notebook

3
ทดสอบถามคำถามจริง

ทดลองถามคำถามบ่อย 20 ข้อ → ตรวจว่าคำตอบถูกต้องหรือไม่

4
เพิ่ม Sources จนตอบได้ครอบคลุม

ถ้าตอบไม่ได้ = ขาดเอกสาร → เพิ่ม Sources ที่เกี่ยวข้อง

5
แชร์ให้ทีมใช้งาน

แชร์ Notebook (Viewer) ให้ทุกคนในทีม พร้อมคำแนะนำวิธีถาม

ภาพ: ตัวอย่างการถาม NotebookLM แล้วได้คำตอบพร้อม citation

9.4 Best Practices การดูแล Notebook ระยะยาว

Notebook ที่ดีต้องได้รับการดูแลอย่างต่อเนื่อง เหมือนกับเว็บไซต์หรือฐานข้อมูลองค์กร

Checklist การดูแล Notebook

ความถี่ สิ่งที่ต้องทำ ผู้รับผิดชอบ
ทุกสัปดาห์ ตรวจว่ามีเอกสารใหม่ที่ต้องเพิ่มหรือไม่ Admin Notebook
ทุกเดือน ลบ Sources ที่เก่า/ยกเลิกแล้ว เพิ่มฉบับใหม่ Admin Notebook
ทุก 3 เดือน ทดสอบถามคำถาม 10 ข้อ ตรวจว่ายังตอบถูกต้อง ทีมงาน
ทุก 6 เดือน ทบทวนโครงสร้าง Notebook ทั้งหมด ปรับหมวดหมู่ถ้าจำเป็น หัวหน้าฝ่าย
เมื่อมีกฎหมายใหม่ อัปโหลดเอกสารใหม่ ลบฉบับเก่าที่ถูกแทนที่ Admin Notebook

Do's and Don'ts

✅ ควรทำ
  • ตั้งชื่อ Notebook ชัดเจน มีวันที่อัปเดต
  • มีคนรับผิดชอบดูแลแต่ละ Notebook
  • บันทึก FAQ/Notes ที่ดีไว้เป็น Reference
  • ทดสอบคำตอบก่อนแชร์ให้ผู้อื่น
  • แจ้งทีมเมื่อเพิ่ม/ลบ Sources
❌ ไม่ควรทำ
  • รวมเอกสารทุกเรื่องไว้ใน Notebook เดียว
  • อัปโหลดเอกสารลับราชการ
  • ปล่อยให้ Sources เก่าค้างไว้โดยไม่อัปเดต
  • ให้สิทธิ์ Editor กับทุกคน (ควร Viewer ก่อน)
  • เชื่อคำตอบ 100% โดยไม่ตรวจ citation
💡 Tips: การตั้งชื่อ Notebook
  • ✅ "[ฝ่ายบริการ] สิทธิ์คนพิการ — อัปเดต มิ.ย. 2569"
  • ✅ "[HR] คู่มือพนักงานใหม่ v2.0"
  • ✅ "[ผู้บริหาร] สรุปผลงาน Q1-2569"
  • ❌ "Notebook 1" / "test" / "เอกสารต่างๆ"

9.5 Lab: สร้าง Knowledge Base 3 หมวด

🧪 Lab ปฏิบัติ: ออกแบบ Knowledge Base สำหรับองค์กร

⏱️ เวลา: 50 นาที | รูปแบบ: กลุ่ม 3-4 คน

📋 โจทย์

ออกแบบและสร้าง Knowledge Base 3 หมวด สำหรับองค์กรเพื่อคนพิการ โดยสร้าง Notebook จริง 3 อัน จัดหมวดหมู่ Sources และทดสอบการใช้งาน

📝 ขั้นตอน
1
วางแผน Knowledge Base (10 นาที)

กำหนดชื่อ 3 Notebooks + รายการ Sources ที่ต้องอัปโหลดในแต่ละอัน

2
สร้าง 3 Notebooks (15 นาที)

สร้าง Notebook ตั้งชื่อให้ชัดเจน → อัปโหลด Sources (อย่างน้อย 3 sources/notebook)

3
ทดสอบถามคำถาม (15 นาที)

ถามคำถาม 5 ข้อ/Notebook → ตรวจว่าตอบได้ถูกต้อง + มี citation

4
สร้าง FAQ สำหรับแต่ละหมวด (5 นาที)

ใช้ Notebook Guide → FAQ → บันทึกเป็น Note

5
นำเสนอ Knowledge Base (5 นาที)

อธิบายโครงสร้าง + Demo ถาม 1 คำถาม/Notebook

📌 ตัวอย่าง 3 หมวดที่แนะนำ
Notebook เนื้อหา Sources ตัวอย่าง
1. สิทธิ์และสวัสดิการ กฎหมาย + สิทธิ์ + การเบิกจ่าย พ.ร.บ. ส่งเสริมฯ, ระเบียบเบี้ยความพิการ
2. คู่มือให้บริการ ขั้นตอน + แบบฟอร์ม + Checklist คู่มือขึ้นทะเบียน, แบบฟอร์ม, Flow
3. รายงาน + แผนงาน สถิติ + ผลงาน + แผนปี รายงานประจำปี, แผนยุทธศาสตร์
✅ เกณฑ์สำเร็จ
  • สร้าง Notebook ครบ 3 อัน ตั้งชื่อชัดเจน
  • แต่ละอันมี Sources อย่างน้อย 3 ชิ้น
  • ทดสอบถามคำถามได้คำตอบถูกต้อง
  • สร้าง FAQ ได้อย่างน้อย 1 หมวด
  • อธิบายได้ว่าจะนำไปใช้งานจริงอย่างไร
💡 Tips สำหรับ Lab
  • แบ่งงานในกลุ่ม — คนละ 1 Notebook ทำพร้อมกัน
  • ใช้เอกสารที่เตรียมมาจากหน่วยงาน (ที่เปิดเผยได้)
  • ถ้าไม่มีเอกสาร ใช้ข้อมูลจากเว็บไซต์ พก. แทนได้
  • หลัง Lab แชร์ Notebook ให้เพื่อนร่วมกลุ่ม (Viewer)
