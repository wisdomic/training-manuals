8.1 ออกแบบระบบด้วย Claude Code

💡 หลักการสำคัญ: ก่อนให้ Claude Code เขียนโค้ด ต้อง "อธิบาย requirement" ให้ชัดเจนก่อน เหมือนสั่งงานลูกทีม ยิ่งบอกละเอียด ผลลัพธ์ยิ่งตรงใจ
วิธีอธิบาย Requirement ให้ Claude Code เข้าใจ

ใช้สูตร WHAT-WHO-HOW:

องค์ประกอบ คำอธิบาย ตัวอย่าง
WHAT — ระบบอะไร ชื่อระบบ + เป้าหมายหลัก "ระบบลงทะเบียนฝึกอบรมออนไลน์"
WHO — ใครใช้ ผู้ใช้ระบบ + สิทธิ์ "ผู้สมัคร (กรอกฟอร์ม) + แอดมิน (ดูข้อมูล)"
HOW — ทำงานอย่างไร เทคโนโลยี + ฟีเจอร์ "ใช้ GAS + Google Sheets, มี CRUD, ส่ง Email"
ตัวอย่าง Prompt: ให้ Claude Code วิเคราะห์ Requirement
🎯 Prompt สำหรับวิเคราะห์ requirement
> ผมต้องการสร้างระบบลงทะเบียนฝึกอบรมสำหรับ สนพ.สุรินทร์
  - ผู้สมัคร: กรอกฟอร์ม (ชื่อ, หน่วยงาน, เบอร์โทร, email, หลักสูตร)
  - แอดมิน: ดูรายชื่อ, อนุมัติ, ส่ง email ยืนยัน
  - เทคโนโลยี: Google Apps Script + Google Sheets
  - ต้องมี: validation, กันซ้ำ, ส่ง email อัตโนมัติ, แจ้ง Line
  
  ช่วยวิเคราะห์ requirement แล้วเสนอ design:
  1. โครงสร้าง Sheets (ต้องมี sheet อะไรบ้าง, คอลัมน์อะไร)
  2. ไฟล์ที่ต้องสร้าง (Code.gs, HTML ต่างๆ)
  3. ฟังก์ชันหลักที่ต้องมี
  4. Flow การทำงาน

📁 ไฟล์ที่ต้องสร้าง:
1. Code.gs — Backend (doGet, doPost, CRUD, Email, Line)
2. index.html — หน้าแรก + ฟอร์มลงทะเบียน
3. admin.html — หน้า Admin ดูข้อมูล/อนุมัติ
4. appsscript.json — ตั้งค่า GAS

⚙️ ฟังก์ชันหลัก:
- doGet(e) → route หน้าต่างๆ
- submitRegistration(data) → บันทึก + ส่ง Email
- getRegistrations() → ดึงข้อมูลสำหรับ Admin
- approveRegistration(regId) → อนุมัติ + แจ้ง Email + Line
- checkDuplicate(email, course) → กันซ้ำ

🔄 Flow:
ผู้สมัคร → กรอกฟอร์ม → Validate → ตรวจซ้ำ → บันทึก Sheets
→ ส่ง Email ยืนยัน → แจ้ง Admin (Line)
→ Admin อนุมัติ → ส่ง Email ยืนยันผู้สมัคร

// ตัวอย่าง validation เลขบัตรประชาชน 13 หลัก
function validateIdCard(id) {
    if (id.length !== 13) return false;
    if (!/^\d{13}$/.test(id)) return false;
    
    // ตรวจ check digit (หลักที่ 13)
    let sum = 0;
    for (let i = 0; i < 12; i++) {
        sum += parseInt(id[i]) * (13 - i);
    }
    let checkDigit = (11 - (sum % 11)) % 10;
    return checkDigit === parseInt(id[12]);
}

// Validate เบอร์โทร
function validatePhone(phone) {
    return /^0[689]\d{8}$/.test(phone.replace(/-/g, ''));
}

// Validate email
function validateEmail(email) {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
}

// ============================================
// ระบบลงทะเบียนฝึกอบรม — สนพ.สุรินทร์
// ============================================

var SHEET_ID = 'YOUR_SHEET_ID_HERE';

// ======== Route ========
function doGet(e) {
  var page = e && e.parameter && e.parameter.page ? e.parameter.page : '';
  
  if (page === 'admin') {
    return HtmlService.createHtmlOutputFromFile('admin')
      .setTitle('Admin - ระบบลงทะเบียน')
      .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
  }
  
  return HtmlService.createHtmlOutputFromFile('index')
    .setTitle('ลงทะเบียนฝึกอบรม - สนพ.สุรินทร์')
    .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
}

function doPost(e) {
  try {
    var data = JSON.parse(e.postData.contents);
    var action = data.action || '';
    
    switch (action) {
      case 'register':
        return _jsonResponse(submitRegistration(data));
      case 'getCourses':
        return _jsonResponse(getCourses());
      case 'getRegistrations':
        return _jsonResponse(getRegistrations());
      case 'approve':
        return _jsonResponse(approveRegistration(data.regId));
      case 'reject':
        return _jsonResponse(rejectRegistration(data.regId, data.reason));
      default:
        return _jsonResponse({ success: false, message: 'Unknown action' });
    }
  } catch (err) {
    return _jsonResponse({ success: false, message: err.toString() });
  }
}

// ======== บันทึกลงทะเบียน ========
function submitRegistration(data) {
  // Validate
  var errors = validateInput(data);
  if (errors.length > 0) {
    return { success: false, errors: errors };
  }
  
  // ตรวจซ้ำ
  if (checkDuplicate(data.email, data.courseId)) {
    return { success: false, message: 'คุณลงทะเบียนหลักสูตรนี้แล้ว' };
  }
  
  // สร้าง ID
  var regId = generateId();
  var now = new Date();
  
  // บันทึก
  var ss = SpreadsheetApp.openById(SHEET_ID);
  var sh = ss.getSheetByName('REGISTRATIONS');
  sh.appendRow([
    regId,
    data.name,
    data.org,
    data.phone,
    data.email,
    data.courseId,
    'pending',
    now.toISOString(),
    ''
  ]);
  
  // ส่ง Email + Line
  sendConfirmEmail({ regId: regId, name: data.name, email: data.email });
  sendLineNotify('📋 ลงทะเบียนใหม่: ' + data.name + ' (' + regId + ')');
  
  return { success: true, regId: regId, message: 'ลงทะเบียนสำเร็จ' };
}

// ======== ช่วยเหลือ ========
function generateId() {
  var chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
  var id = 'REG-';
  for (var i = 0; i < 6; i++) {
    id += chars.charAt(Math.floor(Math.random() * chars.length));
  }
  return id;
}

function _jsonResponse(obj) {
  return ContentService.createTextOutput(JSON.stringify(obj))
    .setMimeType(ContentService.MimeType.JSON);
}

8.5 Debug + แก้ Bug ด้วย Claude Code

💡 Claude Code เก่งมากเรื่อง Debug: แค่บอก "อาการ" ที่เกิดขึ้น Claude จะวิเคราะห์สาเหตุและเสนอวิธีแก้ไขให้
วิธีบอก Bug ให้ Claude Code เข้าใจ

ใช้สูตร SAR (Symptom - Action - Result):

องค์ประกอบ คำอธิบาย ตัวอย่าง
S — Symptom (อาการ) เกิดอะไรขึ้น "ปุ่ม Submit กดแล้วไม่มีอะไรเกิดขึ้น"
A — Action (สิ่งที่ทำ) ทำอะไรก่อนเกิดปัญหา "กรอกฟอร์มครบทุกช่อง แล้วกดปุ่ม"
R — Result (ผลที่คาดหวัง) อยากให้เกิดอะไร "ควรแสดง loading แล้วส่งข้อมูลไป Sheets"
