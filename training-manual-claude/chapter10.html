10.1 เปรียบเทียบ Claude AI vs Co-Work vs Code

💡 สรุป 3 ภาคที่เรียนมา: ตลอด 5 วันที่ผ่านมา เราเรียนรู้ Claude 3 รูปแบบ — แต่ละรูปแบบเหมาะกับงานคนละประเภท
ตารางเปรียบเทียบ 3 ภาค
หัวข้อ ภาค 1: Claude AI (Web) ภาค 2: Claude Co-Work (Kiro) ภาค 3: Claude Code (CLI)
เรียนวันที่ วันที่ 1-2 (12 ชม.) วันที่ 3 (6 ชม.) วันที่ 4-5 (12 ชม.)
ใช้ผ่าน เว็บเบราว์เซอร์ (claude.ai) IDE (Kiro Desktop App) Terminal / Command Line
เหมาะกับ งานทั่วไป ร่างเอกสาร วิเคราะห์ พัฒนาระบบเป็นทีม + workflow เขียนโค้ด + สร้างระบบเร็ว
ใครใช้ ทุกคน (ไม่ต้องมีพื้นฐาน IT) Developer + ทีมพัฒนา Developer ที่ใช้ CLI คล่อง
จุดเด่น ง่าย เข้าใจทันที ใช้ได้เลย Spec-driven, มีแผนก่อนเขียน เร็ว ทรงพลัง ควบคุมได้ละเอียด
ข้อจำกัด ไม่เข้าถึงไฟล์จริง ไม่รันโค้ดได้ ต้องติดตั้ง IDE + เรียนรู้ workflow ต้องใช้ terminal ไม่มี GUI
ราคา ฟรี / $20/เดือน (Pro) ฟรี (Beta) ตาม API usage (~$5-50/เดือน)
อ่าน/เขียนไฟล์ ❌ (แค่ copy-paste) ✅ (ในโปรเจกต์) ✅ (ในโปรเจกต์)
รันคำสั่ง ❌ ✅ ✅
หน้าจอ GUI ✅ (Web) ✅ (IDE) ❌ (Terminal)
ตัวอย่างงาน ร่างหนังสือ สรุปประชุม แปลเอกสาร สร้างระบบ + manage project เขียน GAS, สร้างเว็บ, debug
สรุปแบบง่าย
🌐 Claude AI (Web)

💬

"คุยกับ AI ผ่านเว็บ"

เหมือนมีเลขาส่วนตัว
ช่วยร่าง/แปล/สรุป/วิเคราะห์

ใช้เมื่อ: งานเอกสาร งานวิเคราะห์ งานทั่วไป

🏗️

"ทำงานร่วมกับ AI ใน IDE"

เหมือนมีสถาปนิก

ใช้เมื่อ: พัฒนาระบบใหญ่ ต้องการ workflow

⌨️ Claude Code (CLI)

🚀

"สั่ง AI ใน Terminal"

เหมือนมีโปรแกรมเมอร์
เขียน/แก้/deploy ได้เลย

ใช้เมื่อ: เขียนโค้ด แก้ bug สร้างระบบเร็ว

10.2 เลือกเครื่องมือให้เหมาะกับงาน

Decision Tree: ถ้างาน X → ใช้เครื่องมือ Y
                    ┌─────────────────┐
                    │   งานที่ต้องทำ?   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        ┌──────────┐  ┌──────────┐  ┌──────────┐
        │ งานเอกสาร │  │ พัฒนาระบบ │  │ เขียนโค้ด │
        │ /วิเคราะห์ │  │  ใหญ่/ทีม  │  │  /แก้ bug  │
        └─────┬────┘  └─────┬────┘  └─────┬────┘
              │              │              │
              ▼              ▼              ▼
        ┌──────────┐  ┌──────────┐  ┌──────────┐
        │Claude AI │  │   Kiro    │  │Claude Code│
        │  (Web)   │  │ (Co-Work) │  │  (CLI)   │
        └──────────┘  └──────────┘  └──────────┘
Flowchart: เลือกเครื่องมือตามสถานการณ์
ถ้าคุณต้องการ... → ใช้ เหตุผล
ร่างหนังสือราชการ Claude AI ง่ายที่สุด copy-paste ได้เลย
สรุปเอกสาร PDF 50 หน้า Claude AI อัปโหลดไฟล์ได้ สรุปทันที
วิเคราะห์ข้อมูล Excel Claude AI อัปโหลด + ถามคำถามได้
สร้างเว็บไซต์ 1 หน้า Claude Code สร้างไฟล์จริงได้เลย
สร้างระบบ GAS + Sheets Claude Code เขียน/แก้ไฟล์ + deploy ได้
สร้างระบบใหญ่ (10+ ไฟล์) Kiro มี Spec workflow จัดระบบได้ดี
ทำงานเป็นทีม 3+ คน Kiro มี Steering ควบคุม + ทำงานร่วมกัน
Debug โค้ดที่มี error Claude Code อ่าน error + แก้ไฟล์จริงได้
เขียนบทความ / สื่อประชาสัมพันธ์ Claude AI ถนัดงานเขียน ปรับแต่งง่าย
สร้าง Automation script Claude Code เขียน + รันทดสอบได้ทันที
ใช้ร่วมกัน (Combo) ได้ผลดีที่สุด

Workflow แนะนำ:

  1. Claude AI — วางแผน + ร่าง requirement + ออกแบบ (คุย brainstorm)
  2. Kiro — สร้าง Spec + Requirements + Design (ถ้าเป็นระบบใหญ่)
  3. Claude Code — เขียนโค้ดจริง + deploy + debug
  4. Claude AI — เขียนเอกสาร/คู่มือผู้ใช้

10.3 Workshop: ผู้เข้าอบรมเลือกสร้างงาน/ระบบ

🎯 กิจกรรม (1.5 ชั่วโมง): ผู้เข้าอบรมเลือกโจทย์ 1 ข้อ แล้วสร้างด้วย Claude AI หรือ Claude Code — ทำคนเดียว/ทีม 2-3 คน
รายการฟีเจอร์ 4 ระดับ
🟢 ระดับง่าย (ใช้ Claude AI เป็นหลัก) — 30 นาที
# โจทย์ ใช้เครื่องมือ ผลลัพธ์
1 ร่างหนังสือเชิญวิทยากร (พร้อมใช้) Claude AI เอกสาร Word/PDF
2 สรุปรายงานผลการดำเนินงาน (จาก PDF ที่ให้) Claude AI สรุป 1 หน้า
3 สร้างแบบสอบถาม 20 ข้อ (Google Forms style) Claude AI เนื้อหาคำถาม + ตัวเลือก
4 เขียนโพสต์ประชาสัมพันธ์หลักสูตร (5 โพสต์) Claude AI เนื้อหา Social Media
🔵 ระดับปานกลาง (Claude AI + Artifacts) — 45 นาที
# โจทย์ ใช้เครื่องมือ ผลลัพธ์
5 สร้าง Dashboard HTML แสดงสถิติฝึกอบรม Claude AI (Artifact) หน้าเว็บ + กราฟ
6 สร้างแผนการฝึกอบรม 3 วัน (ตารางเวลาสวยๆ) Claude AI (Artifact) HTML schedule
7 สร้าง Quiz Interactive (10 ข้อ) วัดความรู้ AI Claude AI (Artifact) Web quiz + คะแนน
8 วิเคราะห์ข้อมูล Excel แล้วสร้าง Infographic Claude AI สรุป + Visualization
🟡 ระดับยาก (Claude Code) — 60 นาที
# โจทย์ ใช้เครื่องมือ ผลลัพธ์
9 สร้างเว็บ Portfolio หน่วยงาน (Responsive + Animation) Claude Code เว็บ 1 หน้า
10 สร้างระบบลงชื่อเข้าอบรม (GAS + Sheets) Claude Code ระบบ CRUD
11 สร้าง Multi-step Form + Validation (HTML/JS) Claude Code ฟอร์ม 3 ขั้นตอน
12 สร้าง Admin Dashboard + ตาราง DataTables Claude Code หน้า Admin
🔴 ระดับท้าทาย (Claude Code + Full System) — 90 นาที
# โจทย์ ใช้เครื่องมือ ผลลัพธ์
13 ระบบจองห้องประชุม (ปฏิทิน + ตรวจว่าง + แจ้งเตือน) Claude Code + GAS ระบบเต็ม 3 หน้า
14 ระบบแจ้งซ่อม/IT Helpdesk (ฟอร์ม + Admin + สถานะ + Line) Claude Code + GAS ระบบ + Line Notify
15 ระบบยืม-คืน วัสดุ/ครุภัณฑ์ (Stock + History + Report) Claude Code + GAS ระบบเต็ม + PDF
16 ระบบ KPI Dashboard (ดึง Google Sheets + Chart.js + Auto-update) Claude Code + GAS Dashboard + Report
📌 กฎ Workshop:
  • เลือกโจทย์ตามความสามารถ (ไม่ต้องเลือกยากสุด)
  • ใช้ AI ช่วยได้เต็มที่ (นั่นคือเป้าหมายของหลักสูตร!)
  • ทำทีม 2-3 คนได้ หรือทำคนเดียวก็ได้
  • ไม่ต้องเสร็จ 100% — ให้ demo ได้ก็พอ
  • เตรียมนำเสนอ 5 นาที (ในหัวข้อ 10.4)

10.4 นำเสนอผลงาน

🎤 รูปแบบการนำเสนอ: แต่ละทีม 5 นาที + Q&A 2 นาที = 7 นาทีต่อทีม
Format การนำเสนอ (5 นาที)
ลำดับ เนื้อหา เวลา
1 แนะนำ: ชื่อทีม + โจทย์ที่เลือก + เหตุผล 30 วินาที
2 Demo: แสดงผลงานที่สร้าง (เปิดหน้าจอจริง) 2 นาที
3 Process: ใช้ AI อย่างไร (แสดง prompt ที่ใช้) 1.5 นาที
4 Lessons Learned: เรียนรู้อะไร + ปัญหาที่เจอ 1 นาที
เกณฑ์ประเมิน 5 ด้าน (รวม 20 คะแนน)
ด้าน คะแนน เกณฑ์ 4 คะแนน (ดีเยี่ยม) เกณฑ์ 3 คะแนน (ดี) เกณฑ์ 2 คะแนน (พอใช้)
1. ฟังก์ชัน 4 ทำงานได้ครบ ไม่มี error ทำงานได้ส่วนใหญ่ ทำงานได้บางส่วน
2. UI/UX Design 4 สวยงาม ใช้ง่าย responsive ดูดี เรียบร้อย ใช้งานได้ แต่ไม่สวย
3. ความคิดสร้างสรรค์ 4 มีไอเดียใหม่ ประยุกต์ใช้จริงได้ ต่อยอดจากตัวอย่าง ทำตามโจทย์พื้นฐาน
4. การใช้ AI 4 ใช้ AI อย่างชาญฉลาด prompt ดี ใช้ AI ได้ถูกต้อง ใช้ AI แบบเบื้องต้น
5. การนำเสนอ 4 ชัดเจน น่าสนใจ ตรงเวลา เข้าใจได้ ตรงประเด็น นำเสนอได้ แต่ไม่ชัดเจน
🏆 รางวัล:
  • 🥇 คะแนนรวมสูงสุด — "Best Overall"
  • 🎨 UI/UX สวยที่สุด — "Best Design"
  • 💡 ความคิดสร้างสรรค์ที่สุด — "Most Creative"
  • 🤖 ใช้ AI ได้ดีที่สุด — "Best AI Usage"

10.5 Tips + แหล่งเรียนรู้ + ปิดอบรม

🔟 Top 10 Tips สำหรับใช้ Claude AI ในงานราชการ
  1. เขียน Prompt ให้ชัดเจน — ยิ่งละเอียด ผลลัพธ์ยิ่งตรงใจ ใช้สูตร RCCFO (Role-Context-Constraint-Format-Output)
  2. ใช้ตัวอย่าง (Few-shot) — ให้ตัวอย่างผลลัพธ์ที่ต้องการ AI จะทำตาม pattern ได้ดี
  3. แบ่งงานย่อย — งานใหญ่ แบ่งเป็นหลาย prompt ดีกว่า prompt เดียวยาวมาก
  4. ตรวจสอบทุกครั้ง — AI อาจผิดได้ ตรวจข้อมูลสำคัญก่อนนำไปใช้จริง
  5. ใช้ Projects (claude.ai) — จัดกลุ่มงานเป็น Project เพื่อ AI จำ context ได้
  6. Commit บ่อย (Claude Code) — ทำเสร็จทีละ feature แล้ว commit ย้อนกลับได้
  7. สร้าง CLAUDE.md — กำหนดกฎโปรเจกต์ ไม่ต้องบอกซ้ำทุกครั้ง
  8. อย่ากลัว error — ให้ Claude ช่วย debug ได้เลย วาง error message ใส่แล้วถาม
  9. เรียนรู้จาก output — อ่านโค้ดที่ Claude สร้าง เรียนรู้เทคนิคใหม่ๆ
  10. รักษาความปลอดภัย — ห้ามใส่ข้อมูลลับ (รหัสผ่าน, เลขบัตร) ใน prompt

📝 ใบรับรอง (Certificate)

เงื่อนไขรับใบรับรอง:

  • เข้าเรียนไม่น้อยกว่า 80% ของเวลาทั้งหมด (24 ชม. จาก 30 ชม.)
  • ส่งผลงาน Workshop อย่างน้อย 1 ชิ้น
  • นำเสนอผลงาน (ทีม/เดี่ยว)
  • ทำแบบประเมินความพึงพอใจ

ใบรับรอง: ออกโดย สำนักงานพัฒนาฝีมือแรงงานสุรินทร์ กรมพัฒนาฝีมือแรงงาน กระทรวงแรงงาน

📋 แบบประเมินหลังอบรม

ส่วนที่ 1: ความพึงพอใจ (5 ระดับ)

  1. เนื้อหาหลักสูตรตรงกับความต้องการ
  2. ระยะเวลาเหมาะสม
  3. วิทยากรอธิบายชัดเจน
  4. สื่อการสอนเหมาะสม
  5. สามารถนำไปใช้งานจริงได้

ส่วนที่ 2: ประเมินตนเอง

  1. ก่อนอบรม: ความรู้ด้าน AI (1-10)
  2. หลังอบรม: ความรู้ด้าน AI (1-10)
  3. สิ่งที่จะนำไปใช้ในงานจริง (อธิบาย)
  4. ข้อเสนอแนะ/สิ่งที่ควรปรับปรุง
