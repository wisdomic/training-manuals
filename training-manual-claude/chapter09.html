9.1 Custom Commands — สร้างคำสั่งเอง

9.2 CLAUDE.md — กำหนดกฎโปรเจกต์

├── index.html
└── ...

# CLAUDE.md — กฎโปรเจกต์ระบบลงทะเบียนฝึกอบรม

## ภาพรวม
- ระบบลงทะเบียนฝึกอบรมออนไลน์ สำหรับ สนพ.สุรินทร์
- เทคโนโลยี: Google Apps Script + Google Sheets
- ภาษา: ไทย (comment + UI ทั้งหมด)

## กฎที่ต้องปฏิบัติ (MUST)
1. ❌ ห้ามใช้ template literal (backtick ` `) — GAS ตีความผิดใน HTML
2. ❌ ห้ามใช้ SVG ใน HTML — GAS sanitizer ทำลาย viewBox
3. ❌ ห้ามลบฟังก์ชันเดิมที่ทำงานได้ดีอยู่แล้ว
4. ✅ ใช้ string concatenation เท่านั้น
5. ✅ ทุกฟังก์ชันต้องมี try-catch
6. ✅ ทุก input ต้อง validate ก่อนประมวลผล
7. ✅ comment ภาษาไทย

## โครงสร้างไฟล์
- Code.gs — Backend (doGet, doPost, CRUD, Email, Line)
- index.html — หน้าฟอร์มลงทะเบียน (สาธารณะ)
- admin.html — หน้า Admin
- appsscript.json — ตั้งค่า

## Database (Google Sheets)
- Sheet ID: 1abc...xyz
- REGISTRATIONS: reg_id, name, org, phone, email, course_id, status, reg_date, approved_date
- COURSES: course_id, course_name, start_date, max_seats, current_seats, status
- SETTINGS: key, value

## Coding Standards
- ตั้งชื่อฟังก์ชันเป็น camelCase
- ตั้งชื่อ Sheet เป็น UPPER_CASE
- ใช้ var แทน let/const (compatibility กับ GAS เก่า)
- String: ใช้ single quote เท่านั้น
- ฟังก์ชัน private ขึ้นต้นด้วย _ (เช่น _sendLine, _validate)

## Deploy
- ใช้ clasp push + clasp deploy
- ทุกครั้งที่ deploy ต้องอัปเดต CHANGELOG.md
- Test ก่อน deploy ทุกครั้ง

## Line Notify
- Token ดึงจาก SETTINGS sheet (key: 'line_token')
- User ID: U95dac59d83dc5e26d99914df3ddc27e2
- ส่งแจ้งเตือนเมื่อ: ลงทะเบียนใหม่, อนุมัติ, ปฏิเสธ

## Security
- Rate Limiting: register 3/hr, admin 100/5min
- Input Validation: maxLength 500, ตรวจ XSS
- Password: SHA-256 + Salt (ถ้ามี login)
เปรียบเทียบ CLAUDE.md vs Kiro Steering
คุณสมบัติ CLAUDE.md (Claude Code) Steering Files (Kiro)
ตำแหน่งไฟล์ CLAUDE.md ที่ root .kiro/steering/*.md

เวลาที่อ่าน ทุกครั้งที่เปิด Claude Code เลือก auto/manual inclusion

ความยืดหยุ่น เขียนอิสระ Markdown มีโครงสร้างเฉพาะ (inclusion, globs)
ทำงานร่วมกัน ใส่ใน Git ได้ ทีมใช้ร่วมกัน ใส่ใน Git ได้ ทีมใช้ร่วมกัน
วิธีใช้ CLAUDE.md ให้ได้ผลดี
✅ ควรใส่
  • กฎที่ต้องปฏิบัติเสมอ (MUST/MUST NOT)
  • โครงสร้างโปรเจกต์
  • Coding standards ที่ใช้

9.4 MCP Connections — เชื่อมต่อ API ภายนอก

💡 MCP (Model Context Protocol) คืออะไร: มาตรฐานที่ให้ Claude Code เชื่อมต่อกับเครื่องมือภายนอกได้ เช่น อ่าน/เขียนไฟล์, ค้นหาเว็บ, เชื่อม Database — ขยายความสามารถของ Claude ให้ทำได้มากขึ้น
MCP ทำงานอย่างไร
┌─────────────┐      ┌─────────────┐      ┌─────────────────┐
│  Claude Code │ ←──→ │  MCP Server │ ←──→ │ External Service │
│  (AI Agent)  │      │  (Bridge)   │      │ (File/DB/API)   │
└─────────────┘      └─────────────┘      └─────────────────┘

Claude Code ส่งคำขอ → MCP Server แปลงเป็น API call → ได้ผลลัพธ์กลับ
ตั้งค่า MCP ใน Claude Code

สร้างไฟล์ .claude/mcp.json ในโปรเจกต์:

ตัวอย่าง .claude/mcp.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/project"]
    },
    "web-search": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-server-web-search"],
      "env": {
        "BRAVE_API_KEY": "your-brave-api-key"
      }
    },
    "google-sheets": {
      "command": "npx",
      "args": ["-y", "mcp-server-google-sheets"],
      "env": {
        "GOOGLE_SERVICE_ACCOUNT_KEY": "./credentials/service-account.json"
      }
    }
  }
}
MCP Servers ที่น่าใช้
MCP Server ความสามารถ ประโยชน์สำหรับงานราชการ
filesystem อ่าน/เขียน/ค้นหาไฟล์ จัดการไฟล์โปรเจกต์
web-search ค้นหาข้อมูลจากเว็บ หาข้อมูลระเบียบ กฎหมาย อ้างอิง
google-sheets อ่าน/เขียน Google Sheets จัดการข้อมูลใน Sheets โดยตรง
google-drive จัดการไฟล์ใน Drive อัปโหลด/ดาวน์โหลดเอกสาร
sqlite จัดการ SQLite database ระบบฐานข้อมูลขนาดเล็ก
github จัดการ GitHub repos สร้าง PR, review code, issues

9.5 ใช้ Claude Code กับ Docker/Cloud

💡 ภาพรวม: Claude Code สามารถช่วยเขียน Dockerfile, docker-compose, และ deploy ขึ้น Cloud ได้ — เหมาะสำหรับงานที่ต้อง deploy ระบบจริง
Docker Basics สำหรับ GAS Developer
Concept อธิบายง่ายๆ เปรียบเทียบ
Docker Image "แม่พิมพ์" ของแอป (ระบบ + โค้ด + dependencies) เหมือน ISO ของ Windows
Container "แอปที่กำลังรัน" จาก Image เหมือนเปิด Virtual Machine
Dockerfile สูตรสร้าง Image (step-by-step) เหมือน recipe อาหาร
docker-compose รัน Container หลายตัวพร้อมกัน เหมือนสั่งอาหาร set
ตัวอย่าง: ใช้ Claude Code สร้าง Docker Setup
Prompt: สร้าง Dockerfile สำหรับ Node.js + clasp
> สร้าง Dockerfile สำหรับ development environment ที่มี:
  - Node.js 20
  - @google/clasp (GAS CLI tool)
  - Git
  - Claude Code (@anthropic-ai/claude-code)
  - Working directory: /app

Claude:
✏️  Creating: Dockerfile

FROM node:20-slim

# ติดตั้ง Git
RUN apt-get update && apt-get install -y git && rm -rf /var/lib/apt/lists/*

# ติดตั้ง global tools
RUN npm install -g @google/clasp @anthropic-ai/claude-code

# Working directory
WORKDIR /app

# Mount volume สำหรับ source code
VOLUME ["/app"]

# Default command
CMD ["bash"]

✏️  Creating: docker-compose.yml

version: '3.8'
services:
  dev:
    build: .
    volumes:
      - .:/app
      - ~/.clasprc.json:/root/.clasprc.json
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
    stdin_open: true
    tty: true

✅ Created Dockerfile + docker-compose.yml
