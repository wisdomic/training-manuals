7.1 Claude Code คืออะไร — CLI tool สำหรับเขียนโค้ด

💡 สรุปง่ายๆ: Claude Code เป็นเครื่องมือที่ให้คุณ "คุยกับ AI ในเทอร์มินัล" เพื่อให้ AI ช่วยเขียนโค้ด อ่านไฟล์ แก้ไขไฟล์ และรันคำสั่งต่างๆ ได้โดยตรง
Claude Code คืออะไร?

Claude Code เป็น AI coding assistant แบบ command-line (CLI) ที่พัฒนาโดย Anthropic บริษัทผู้สร้าง Claude สามารถ:

✅ สิ่งที่ Claude Code ทำได้
  • อ่านไฟล์โค้ดแล้ววิเคราะห์/อธิบาย
  • เขียนโค้ดใหม่ตามที่สั่ง
  • แก้ไขไฟล์ที่มีอยู่แล้ว
  • รันคำสั่ง terminal
  • ค้นหาไฟล์ในโปรเจกต์
  • หา bug และแก้ไข
  • Refactor โค้ด
  • เขียน unit tests
  • สร้าง Git commits
  • อธิบายโค้ดเป็นภาษาไทย
📝 ลักษณะเฉพาะ
  • ทำงานใน Terminal — ไม่มีหน้าจอ GUI
  • อ่าน/เขียนไฟล์จริง — แก้ไขโค้ดของเราได้
  • เข้าถึง file system — ดูโครงสร้างโปรเจกต์ได้ทั้งหมด
  • รันคำสั่งได้ — npm, git, python ฯลฯ
  • ใช้ API Key — เสียค่าใช้จ่ายตามการใช้งาน
  • ทำงาน local — ไฟล์อยู่ในเครื่องเรา
  • รองรับทุกภาษา — JS, Python, Go, Rust ฯลฯ
  • ไม่ต้องใช้ IDE — ใช้ terminal อย่างเดียว
Claude Code ต่างจาก claude.ai อย่างไร?
คุณสมบัติ claude.ai (Web) Claude Code (CLI)
ใช้งานผ่าน เว็บเบราว์เซอร์ Terminal / Command Line
อ่าน/เขียนไฟล์ ❌ ไม่ได้โดยตรง ✅ อ่าน/เขียน/ลบ ได้เลย
รันคำสั่ง ❌ ไม่ได้ ✅ รัน terminal commands ได้
เหมาะสำหรับ งานทั่วไป ร่างเอกสาร วิเคราะห์ เขียนโค้ด พัฒนาระบบ
ค่าใช้จ่าย $20/เดือน (Pro) หรือฟรี จ่ายตาม API usage
ต้องติดตั้ง ไม่ต้อง ต้องติดตั้ง Node.js + npm
⚠️ หมายเหตุ: Claude Code เหมาะสำหรับนักพัฒนา/โปรแกรมเมอร์ที่คุ้นเคยกับ terminal ผู้ที่ไม่คุ้นเคยอาจใช้ Kiro หรือ Cursor ซึ่งมีหน้าจอ GUI แทน (จะกล่าวในหัวข้อ 7.5)

7.2 ติดตั้ง Claude Code

สิ่งที่ต้องมีก่อนติดตั้ง (Prerequisites)
รายการ เวอร์ชันขั้นต่ำ วิธีตรวจสอบ
Node.js v18.0.0 ขึ้นไป node --version
npm v9.0.0 ขึ้นไป npm --version
ระบบปฏิบัติการ Windows 10+, macOS 12+, Linux -
Anthropic API Key - สมัครที่ console.anthropic.com
ขั้นตอนที่ 1: ติดตั้ง Node.js (ถ้ายังไม่มี)
1 ดาวน์โหลด Node.js

ไปที่ https://nodejs.org แล้วดาวน์โหลดเวอร์ชัน LTS (Long Term Support)

2 ติดตั้ง

Double-click ไฟล์ที่ดาวน์โหลด แล้ว Next → Next → Install → Finish

3 ตรวจสอบ

เปิด Terminal/Command Prompt แล้วพิมพ์:

node --version
# ควรแสดง v18.x.x หรือสูงกว่า

npm --version
# ควรแสดง v9.x.x หรือสูงกว่า
ขั้นตอนที่ 2: ติดตั้ง Claude Code
# ติดตั้ง Claude Code แบบ global
npm install -g @anthropic-ai/claude-code

# ตรวจสอบว่าติดตั้งสำเร็จ
claude --version
💡 สำหรับ Windows: ใช้ PowerShell หรือ Command Prompt (Run as Administrator) ในการติดตั้ง ถ้าเจอ error เรื่อง permission ให้ใช้คำสั่ง: 
npm install -g @anthropic-ai/claude-code --force
⚠️ ปัญหาที่พบบ่อย:
  • 'npm' is not recognized → ต้องติดตั้ง Node.js ก่อน หรือเปิด Terminal ใหม่
  • EACCES permission denied → ใช้ sudo (Mac/Linux) หรือ Run as Admin (Windows)
  • node version too old → อัปเดต Node.js เป็นเวอร์ชัน 18+

7.3 ตั้งค่า API Key + เริ่มใช้งาน

ขั้นตอนที่ 1: สมัคร Anthropic API Key
1 เข้า Anthropic Console

ไปที่ https://console.anthropic.com

2 สมัครบัญชี / Login

ใช้ email สมัคร หรือ login ด้วย Google

3 สร้าง API Key

ไปที่ Settings → API Keys → Create Key

ตั้งชื่อ Key เช่น "claude-code-training" แล้วคัดลอก Key ที่ได้ (ขึ้นต้นด้วย sk-ant-...)

4 เติมเงิน (Credits)

ไปที่ Billing → Add credits เติมขั้นต่ำ $5 (ประมาณ 175 บาท)

🚨 สำคัญ: เก็บ API Key ไว้อย่างปลอดภัย ห้ามแชร์ ห้ามวางใน public repository ถ้าหลุดออกไป ให้ revoke ทันที
ขั้นตอนที่ 2: ตั้งค่า API Key
Windows (PowerShell)
# ตั้งค่าชั่วคราว (หายเมื่อปิด terminal)
$env:ANTHROPIC_API_KEY = "sk-ant-api03-xxxxx..."

# ตั้งค่าถาวร
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-api03-xxxxx...", "User")
macOS / Linux
# ตั้งค่าชั่วคราว
export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx..."

# ตั้งค่าถาวร (เพิ่มในไฟล์ ~/.bashrc หรือ ~/.zshrc)
echo 'export ANTHROPIC_API_KEY="sk-ant-api03-xxxxx..."' >> ~/.zshrc
source ~/.zshrc
ขั้นตอนที่ 3: เริ่มใช้งาน
# เข้า folder โปรเจกต์ที่ต้องการ
cd C:\MyProject

# เริ่ม Claude Code
claude

# Claude Code จะแสดงหน้าจอพร้อมรับคำสั่ง
# พิมพ์สิ่งที่ต้องการแล้วกด Enter
💰 ค่าใช้จ่าย API Pricing
Model Input (ต่อ 1M tokens) Output (ต่อ 1M tokens) ประมาณค่าใช้จ่าย
Claude Sonnet 4 $3 $15 ~฿1-5 ต่อคำสั่ง
Claude Opus 4 $15 $75 ~฿5-20 ต่อคำสั่ง
Claude Haiku 3.5 $0.80 $4 ~฿0.3-2 ต่อคำสั่ง
💡 เคล็ดลับประหยัด:
  • ใช้ Sonnet เป็นหลัก (คุ้มค่าที่สุด)
  • คำสั่งยาวๆ ละเอียดจะแพงกว่า → พิมพ์กระชับ
  • ใช้ /compact เพื่อลดขนาด context
  • ตั้ง spending limit ที่ console เพื่อควบคุมค่าใช้จ่าย
  • $5 (175 บาท) ใช้ได้ประมาณ 1-2 สัปดาห์สำหรับการเรียนรู้

7.4 คำสั่งพื้นฐาน — สร้างไฟล์, แก้ไข, อ่านโค้ด

เมื่อเข้า interactive mode จะเห็น prompt > ให้พิมพ์คำสั่งภาษาไทยหรืออังกฤษได้เลย

ตัวอย่างคำสั่งที่ใช้บ่อย
📁 สร้างไฟล์ใหม่

  แสดงข้อมูลหลักสูตรฝึกอบรม 3 หลักสูตร ใช้ Bootstrap 5

Claude: ผมจะสร้างไฟล์ index.html ให้ครับ
✏️  Creating file: index.html
[...แสดงเนื้อหาไฟล์...]
✅ Created index.html (45 lines)
📖 อ่านและอธิบายโค้ด
> อ่าน Code.gs แล้วอธิบายว่าฟังก์ชัน doGet ทำอะไร

Claude: ผมจะอ่านไฟล์ Code.gs ให้ครับ
📖 Reading: Code.gs

ฟังก์ชัน doGet ทำหน้าที่:
1. รับ request จากผู้ใช้ (parameter: page)
2. ตรวจสอบว่าเป็นหน้าไหน (admin/user/form)
3. สร้าง HtmlOutput ตาม template ที่กำหนด
4. return หน้า HTML กลับไปให้ browser

โครงสร้าง:
- ถ้า page = "admin" → แสดง admin.html
- ถ้า page = "form" → แสดง user-form.html  
- ถ้าไม่ระบุ → แสดง index.html (หน้าแรก)
🐛 หา Bug และแก้ไข
> อ่านไฟล์ script.js แล้วหา bug ที่ทำให้ปุ่ม submit ไม่ทำงาน

Claude: ผมจะตรวจสอบไฟล์ script.js ให้ครับ
📖 Reading: script.js

🐛 พบปัญหา:
1. บรรทัดที่ 23: ใช้ getElementById("submitBtn") 
   แต่ใน HTML ตั้ง id="submit-btn" (มี hyphen)
   
2. บรรทัดที่ 45: addEventListener ถูกเรียก
   ก่อน DOM load เสร็จ

💡 แก้ไข:
- เปลี่ยน "submitBtn" → "submit-btn"  
- ย้าย script ไปไว้ก่อน </body> หรือใช้ DOMContentLoaded

ต้องการให้รัน server ทดสอบไหมครับ?
Slash Commands ที่สำคัญ
คำสั่ง ความหมาย ใช้เมื่อ
/help แสดงรายการคำสั่งทั้งหมด เมื่อจำคำสั่งไม่ได้
/clear ล้างประวัติสนทนาทั้งหมด เริ่มใหม่ เมื่อต้องการเริ่มงานใหม่
/compact สรุป context ให้สั้นลง ประหยัด token เมื่อสนทนายาวเกินไป
/model เปลี่ยน model (sonnet/opus/haiku) เมื่อต้องการประหยัดหรือใช้ model ที่ดีกว่า
/cost แสดงค่าใช้จ่ายของ session นี้ ตรวจสอบยอดใช้จ่าย
/permissions ดู/ตั้งค่าสิทธิ์ (อ่าน/เขียน/รัน) ควบคุมสิ่งที่ Claude ทำได้
Ctrl+C หยุดคำสั่งที่กำลังรัน เมื่อต้องการยกเลิก
exit หรือ Ctrl+D ออกจาก Claude Code เลิกใช้งาน

💡 เคล็ดลับ:
  • พิมพ์ภาษาไทยได้เลย Claude เข้าใจ
  • ยิ่งบอกรายละเอียดมาก ผลลัพธ์ยิ่งตรง
  • ถ้าไม่พอใจ ให้บอก "แก้ไขตรงนี้..." Claude จะปรับให้

7.5 เปรียบเทียบ Claude Code vs Kiro vs Cursor vs GitHub Copilot

ปัจจุบันมี AI coding tools หลายตัว แต่ละตัวเหมาะกับการใช้งานต่างกัน:

คุณสมบัติ Claude Code Kiro (AWS) Cursor GitHub Copilot
ประเภท CLI (Terminal) IDE (VS Code-based) IDE (VS Code-based) Extension (VS Code/JetBrains)
บริษัทผู้พัฒนา Anthropic Amazon (AWS) Anysphere GitHub / Microsoft
AI Model Claude (Sonnet/Opus) Claude + หลาย Model Claude + GPT + หลาย Model GPT-4o / Claude
ราคา API usage (~$5-50/เดือน) ฟรี (ช่วง Beta) $20/เดือน (Pro) $10-19/เดือน
ติดตั้ง npm install ดาวน์โหลด IDE ดาวน์โหลด IDE ติดตั้ง Extension
อ่าน/เขียนไฟล์ ✅ ✅ ✅ ✅ (จำกัด)
รันคำสั่ง ✅ ✅ ✅ ❌
Auto-complete ❌ ✅ ✅ ✅ (จุดเด่น)
Multi-file Edit ✅ ✅ ✅ จำกัด
Spec-driven ❌ ✅ (จุดเด่น) ❌ ❌
หน้าจอ GUI ❌ (Terminal only) ✅ (Full IDE) ✅ (Full IDE) ✅ (ใน VS Code)
เหมาะกับ Developer ใช้ CLI คล่อง ทีมพัฒนา ต้องการ workflow ดี Developer ทั่วไป เขียนโค้ดเร็วขึ้น
จุดเด่นของแต่ละตัว
⌨️ Claude Code

จุดเด่น:

  • เบา ไม่ต้องติดตั้ง IDE
  • ทำงานได้ทุกที่ที่มี terminal
  • ควบคุมได้ละเอียด (permissions)
  • ใช้ Claude model ตัวล่าสุดเสมอ

เหมาะกับ: โปรแกรมเมอร์ที่ชอบ terminal, ใช้ SSH, server management

🚀 Kiro (AWS)

จุดเด่น:

  • Spec-driven development (วางแผนก่อนเขียน)
  • Steering files ควบคุม AI
  • ฟรี (ช่วง Beta)
  • ใช้ Claude + รองรับหลาย model

เหมาะกับ: ทีมพัฒนาที่ต้องการ workflow เป็นระบบ, งานระยะยาว

🖱️ Cursor

จุดเด่น:

  • GUI สวย ใช้ง่าย
  • เลือก AI model ได้หลายตัว
  • Chat + Composer + Auto-complete
  • Community ใหญ่

เหมาะกับ: Developer ทั่วไป ที่อยากได้ AI IDE ครบวงจร

🤖 GitHub Copilot

จุดเด่น:

  • Auto-complete ดีที่สุด (inline suggestion)
  • ใช้ใน VS Code / JetBrains ได้เลย
  • ราคาถูก ($10/เดือน)
  • ไม่ต้องเปลี่ยน IDE

เหมาะกับ: คนที่ใช้ VS Code อยู่แล้ว ต้องการ autocomplete เร็วขึ้น

แนะนำตามระดับผู้ใช้
ระดับ แนะนำ เหตุผล
เริ่มต้น (ไม่เคยเขียนโค้ด) claude.ai (Web) + Kiro ใช้ง่าย มีหน้าจอ ไม่ต้องตั้งค่าเยอะ
กลาง (เขียนโค้ดได้บ้าง) Cursor หรือ Kiro + Copilot GUI ช่วยได้เยอะ + autocomplete เร็ว
สูง (Developer เต็มตัว) Claude Code + Kiro ควบคุมได้ละเอียด ทำงานเร็ว flexible
ทีม (ทำงานหลายคน) Kiro + Claude Team Spec-driven workflow + แชร์ Projects

แบบฝึกหัด 7.3 (สร้างไฟล์): ใช้ Claude Code สร้างไฟล์ต่อไปนี้:

  1. สร้าง greeting.html — หน้าเว็บแนะนำตัวเอง (ใช้ Bootstrap 5)
  2. สร้าง calculator.js — ฟังก์ชันคำนวณ บวก ลบ คูณ หาร
  3. สร้าง budget.py — คำนวณงบประมาณฝึกอบรม

แบบฝึกหัด 7.4 (อ่านโค้ด): ให้ Claude Code อ่านไฟล์ในโปรเจกต์จริง แล้ว:

  • อธิบายว่าฟังก์ชันหลักทำอะไร
  • หาจุดที่อาจเป็น bug
  • แนะนำการปรับปรุง

แบบฝึกหัด 7.5 (เปรียบเทียบ): ลองใช้ทั้ง Claude Code (CLI) และ claude.ai (Web) ทำงานเดียวกัน:

  • สั่ง: "สร้าง HTML table แสดงหลักสูตรฝึกอบรม 5 หลักสูตร"
  • เปรียบเทียบ: ตัวไหนสะดวกกว่า? ผลลัพธ์ต่างกันไหม?
  • สรุป: สถานการณ์ไหนควรใช้ตัวไหน?
🎯 สิ่งที่ควรได้จากบทนี้:
  1. เข้าใจว่า Claude Code คืออะไร ต่างจาก claude.ai อย่างไร
  2. ติดตั้งและตั้งค่า Claude Code ได้
  3. ใช้คำสั่งพื้นฐาน: สร้างไฟล์, อ่านโค้ด, แก้ bug, รันคำสั่ง
  4. เลือกเครื่องมือ AI coding ที่เหมาะสมกับตนเองได้
