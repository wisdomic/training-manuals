6.1 Claude Team Plan — แชร์ Projects กับทีม

💡 สรุปง่ายๆ: Team Plan ช่วยให้ทีมใช้ Claude ร่วมกัน แชร์ Projects ข้อมูลอ้างอิง และ Prompt ได้ โดยมี Admin จัดการสิทธิ์
Team Plan คืออะไร?

Claude Team Plan เป็นแพ็กเกจสำหรับองค์กร ราคา $30/คน/เดือน (ขั้นต่ำ 5 คน) ที่เพิ่มความสามารถในการ:

  • แชร์ Projects กับสมาชิกในทีม
  • จัดการสิทธิ์ Admin / Member
  • ใช้ Claude ได้มากกว่า Pro Plan ถึง 2 เท่า
  • ข้อมูลไม่ถูกนำไปฝึก AI (Data privacy)
เปรียบเทียบ Pro vs Team Plan
ฟีเจอร์ Pro ($20/เดือน) Team ($30/คน/เดือน)
จำนวนผู้ใช้ 1 คน 5 คนขึ้นไป
ปริมาณใช้งาน สูง สูงกว่า 2 เท่า
Projects ✅ ส่วนตัว ✅ ส่วนตัว + แชร์ทีม
แชร์ Projects ❌ ✅
Admin Console ❌ ✅
Data Privacy ไม่นำไปฝึก ไม่นำไปฝึก + Admin controls
Billing บุคคล รวม (จ่ายทีเดียว)
สิทธิ์ Admin vs Member
👑 Admin
  • เพิ่ม/ลบสมาชิกทีม
  • จัดการ Billing
  • ตั้งค่า Team settings
  • ดูรายงานการใช้งาน
  • สร้าง/แชร์ Projects
  • กำหนดนโยบาย AI ของทีม
👤 Member
  • ใช้งาน Claude ได้ตามปกติ
  • เข้าถึง Shared Projects ของทีม
  • สร้าง Projects ส่วนตัวได้
  • แชร์ Conversations ได้
  • ❌ ไม่สามารถจัดการสมาชิก
  • ❌ ไม่สามารถดู Billing
วิธีแชร์ Project กับทีม
  1. เปิด Project ที่ต้องการแชร์
  2. คลิก Share หรือไอคอนแชร์
  3. เลือก Share with team
  4. สมาชิกทุกคนในทีมจะเห็น Project นี้ในรายการ
💡 ข้อดีสำหรับราชการ: ทีม 5-10 คนที่ทำงานด้วยกัน สามารถแชร์ Knowledge Base เดียวกัน เช่น ระเบียบ คู่มือ FAQ ทำให้ทุกคนตอบคำถามได้ตรงกัน ลดความผิดพลาด

6.2 Shared Conversations — สนทนาร่วมกัน

นอกจากแชร์ Projects แล้ว ยังสามารถแชร์ บทสนทนาเฉพาะ กับเพื่อนร่วมงานได้ เหมือนส่งลิงก์ Line ให้คนอื่นอ่าน

วิธีแชร์ Conversation
  1. เปิด Conversation ที่ต้องการแชร์
  2. คลิกไอคอน Share (มุมขวาบนของ chat)
  3. เลือก Create share link
  4. คัดลอกลิงก์ แล้วส่งให้เพื่อนร่วมงาน
ใช้เมื่อไร?
✅ เหมาะสำหรับ
  • แชร์ผลลัพธ์ที่ Claude ช่วยสร้าง เช่น ร่างเอกสาร, สรุปข้อมูล
  • ให้เพื่อนร่วมงานดูวิธีที่เราถาม Claude (เรียนรู้ Prompt)
  • ส่งงานให้หัวหน้าตรวจทาน
  • เก็บเป็นตัวอย่างสำหรับทีม
❌ ไม่เหมาะสำหรับ
  • Conversation ที่มีข้อมูลส่วนบุคคล
  • Conversation ที่มีข้อมูลลับราชการ
  • ลิงก์อาจถูกส่งต่อให้คนนอกองค์กร
  • ไม่เหมาะกับข้อมูลที่เปลี่ยนแปลงบ่อย
ข้อดี / ข้อเสียของ Shared Conversations
ข้อดี ข้อเสีย
ง่าย ส่งลิงก์ให้ใครก็ได้ คนที่ได้ลิงก์ต้องมีบัญชี Claude
เก็บประวัติสนทนาทั้งหมด ไม่สามารถแก้ไขหลังแชร์
ผู้รับสามารถ "ต่อยอด" สนทนาได้ ต้องระวังข้อมูลส่วนตัว
ดีสำหรับแชร์ best practices ลิงก์ไม่หมดอายุ (ต้องลบเอง)

6.3 Prompt Library — รวม Prompt ที่ใช้บ่อย

Prompt Library คือการรวบรวม Prompt Templates ที่ทีมใช้บ่อยไว้ด้วยกัน เพื่อให้ทุกคนใช้ได้ทันที ไม่ต้องคิดใหม่ทุกครั้ง

💡 แนวคิด: สร้าง "คลัง Prompt" ที่ทุกคนในทีมเข้าถึงได้ เหมือนมี template สำเร็จรูป แค่เปลี่ยนข้อมูลก็ใช้ได้เลย
วิธีสร้าง Prompt Library
  1. รวบรวม Prompt ที่แต่ละคนใช้บ่อยและได้ผลดี
  2. จัดหมวดหมู่ตามประเภทงาน
  3. เก็บไว้ใน Google Docs / Sheets ที่ทีมเข้าถึงได้
  4. หรือใส่เป็น Text content ใน Shared Project
📚 ตัวอย่าง Prompt Library สำหรับงานราชการ

6.4 Best Practices — กฎการใช้ AI ในองค์กร

การใช้ AI ในองค์กรราชการ จำเป็นต้องมี นโยบาย (AI Policy) ที่ชัดเจน เพื่อป้องกันความเสี่ยงและใช้ประโยชน์อย่างเหมาะสม

หลักการ 5 ข้อ สำหรับนโยบาย AI ที่ดี

🔍

1. ตรวจสอบเสมอ

ผลลัพธ์จาก AI ต้องผ่านการตรวจสอบจากมนุษย์ก่อนใช้งานจริง ห้ามเชื่อ 100%

🔒

2. ไม่ใส่ข้อมูลลับ

ห้ามใส่ข้อมูลลับราชการ ข้อมูลส่วนบุคคล หรือข้อมูลที่ยังไม่เปิดเผย

📝

3. อ้างอิง AI

เมื่อใช้ AI ช่วยผลิตผลงาน ควรระบุว่าใช้ AI ช่วย (โดยเฉพาะงานเผยแพร่)

👤

4. Human-in-the-Loop

มนุษย์ต้องเป็นผู้ตัดสินใจสุดท้าย AI เป็นเพียงผู้ช่วย ไม่ใช่ผู้ตัดสินใจ

📊

5. รับผิดชอบ

ผู้ใช้ AI รับผิดชอบผลลัพธ์ ไม่ว่า AI จะสร้างอะไรมา คนเป็นผู้รับผิดชอบ

⚖️

6. เป็นธรรม

ระวัง bias ของ AI ตรวจสอบว่าผลลัพธ์ไม่เลือกปฏิบัติหรือลำเอียง

ตัวอย่างนโยบาย AI สำหรับหน่วยงานราชการ
📋 นโยบายการใช้ AI — สำนักงานพัฒนาฝีมือแรงงานสุรินทร์
  1. การใช้ AI ที่อนุญาต:
    • ร่างเอกสาร หนังสือราชการ รายงาน (ต้องตรวจทานก่อนส่ง)
    • วิเคราะห์ข้อมูลสถิติ (ต้องตรวจสอบตัวเลขก่อนใช้)
    • สรุปเอกสาร บันทึกการประชุม
    • สร้างสื่อประชาสัมพันธ์
    • แปลภาษา ตรวจคำผิด
  2. การใช้ AI ที่ห้าม:
    • ห้ามใส่ข้อมูลส่วนบุคคลของประชาชน
    • ห้ามใส่ข้อมูลลับราชการ
    • ห้ามใช้ AI ตัดสินใจแทนผู้มีอำนาจ
    • ห้ามเผยแพร่ผลงาน AI โดยไม่ตรวจสอบ
  3. ขั้นตอนการตรวจสอบ:
    • ผลลัพธ์จาก AI ต้องผ่านการตรวจจากเจ้าของงาน
    • เอกสารสำคัญ ต้องผ่านหัวหน้างานก่อนเผยแพร่
    • ตัวเลข/สถิติ ต้อง cross-check กับแหล่งข้อมูลจริง
⚠️ สำคัญ — Human-in-the-Loop: AI เป็นเครื่องมือช่วย ไม่ใช่ผู้ตัดสินใจ ทุกครั้งที่ใช้ AI ในงานราชการ ผู้รับผิดชอบต้องตรวจสอบความถูกต้อง ตรวจทานข้อมูล และลงนามรับรองก่อนส่งงาน

6.5 ความปลอดภัย — ข้อมูลที่ห้ามใส่, ข้อจำกัด

🚨 คำเตือนสำคัญ: ข้อมูลที่ใส่ใน Claude จะถูกส่งไปประมวลผลบน Cloud Server ของ Anthropic (สหรัฐอเมริกา) แม้จะมีนโยบายไม่นำข้อมูลไปฝึก AI แต่ก็ต้องระมัดระวังข้อมูลที่ใส่
ตาราง Do / Don't — ข้อมูลที่ใส่ได้ vs ห้ามใส่
✅ ใส่ได้ (Do) ❌ ห้ามใส่ (Don't)
ข้อมูลทั่วไปที่เปิดเผยได้ เลขบัตรประชาชน ของประชาชน/บุคลากร
ระเบียบ กฎหมาย หนังสือเวียน (เปิดเผย) ข้อมูลส่วนบุคคล ชื่อ-นามสกุล + ที่อยู่ + โทรศัพท์ ของผู้อื่น
สถิติที่เผยแพร่แล้ว ข้อมูลลับราชการ ที่มีชั้นความลับ
ร่างเอกสาร (ไม่มีข้อมูลส่วนบุคคล) รหัสผ่าน ทุกประเภท
คำถามทั่วไปเกี่ยวกับระเบียบ ข้อมูลทางการเงิน เลขบัญชีธนาคารของผู้อื่น
ข้อมูลสถิติรวม (ไม่ระบุตัวบุคคล) ข้อมูลสุขภาพ ประวัติการรักษาพยาบาล
ตัวอย่างเอกสาร (ลบชื่อ/ข้อมูลจริงออก) เอกสารสัญญา ที่ยังไม่ลงนาม/ยังไม่เปิดเผย
คำถามความรู้ทั่วไป ข้อสอบ/ข้อมูลการสอบ ที่ยังไม่เปิดเผย
โค้ดทั่วไป ไม่มี credentials API Keys, Tokens, Secrets
แผนงานทั่วไปที่เปิดเผยได้ ข้อมูลการสอบสวน ทางวินัย
เทคนิคป้องกัน — ทำอย่างไรเมื่อต้องใช้ข้อมูลจริง
🔄 วิธีที่ 1: ใช้ข้อมูลสมมติ
❌ แบบผิด:
"ร่างหนังสือถึง นายสมชาย ใจดี 
เลขบัตร 1-3201-xxxxx-xx-x"

✅ แบบถูก:
"ร่างหนังสือถึง [ชื่อผู้รับ] 
เลขบัตร [เลขบัตรประชาชน]"

ใช้ placeholder แทนข้อมูลจริง แล้วเติมข้อมูลจริงตอนสุดท้าย

🔄 วิธีที่ 2: ลบข้อมูลระบุตัว
❌ แบบผิด:
"วิเคราะห์ข้อมูลผู้เข้าอบรม:
1. นายสมชาย ใจดี อายุ 35 
2. นางสาวสมหญิง รักดี อายุ 28"

✅ แบบถูก:
"วิเคราะห์ข้อมูลผู้เข้าอบรม:
1. ผู้เข้าอบรมคนที่ 1 อายุ 35 
2. ผู้เข้าอบรมคนที่ 2 อายุ 28"

ลบชื่อจริง ใช้เลขหรือรหัสแทน

ข้อจำกัดของ Claude ที่ต้องรู้
ข้อจำกัด รายละเอียด วิธีแก้
ข้อมูลไม่ real-time Claude ไม่ได้เชื่อมต่ออินเทอร์เน็ต ไม่รู้เหตุการณ์ปัจจุบัน ใส่ข้อมูลล่าสุดใน Project / วาง text ให้
Hallucination อาจแต่งข้อมูล ตัวเลข อ้างอิงที่ไม่มีจริง ตรวจสอบข้อเท็จจริงเสมอ อย่าเชื่อตัวเลข 100%
Context จำกัด จำข้อมูลได้แค่ในบทสนทนา ไม่จำข้ามครั้ง ใช้ Projects เก็บข้อมูลถาวร
ไม่มีความเห็นส่วนตัว Claude ไม่สามารถ "ตัดสินใจ" แทนคนได้ ให้ AI เสนอทางเลือก คนเป็นผู้ตัดสิน
ภาษาไทยอาจคลาดเคลื่อน บางครั้งใช้คำที่ฟังไม่เป็นธรรมชาติ ตรวจทาน แก้ไขภาษาก่อนส่ง

แบบฝึกหัด 6.1: ร่างนโยบาย AI Policy สำหรับหน่วยงานของท่าน (อ้างอิงจากตัวอย่างข้างต้น) ไม่น้อยกว่า 10 ข้อ

แบบฝึกหัด 6.2: สร้าง Prompt Library สำหรับงานที่ท่านทำบ่อย 5 prompt โดย:

  • ระบุชื่อ Prompt
  • เขียน template (มี placeholder สำหรับเปลี่ยนข้อมูล)
  • ระบุตัวอย่างการใช้งาน 1 ตัวอย่าง

แบบฝึกหัด 6.3: จากข้อมูลสมมติต่อไปนี้ ให้ระบุว่าอันไหน ✅ ใส่ได้ / ❌ ห้ามใส่:

  1. "สรุปจำนวนผู้ผ่านการฝึกอบรมปี 2568 จำนวน 1,500 คน" — [ตอบ: ?]
  2. "นายสมชาย ใจดี เลขบัตร 1-3201-00001-00-1 ผ่านการทดสอบ" — [ตอบ: ?]
  3. "ระเบียบกรมพัฒนาฝีมือแรงงาน ว่าด้วยค่าใช้จ่ายฯ พ.ศ. 2563" — [ตอบ: ?]
  4. "รหัสผ่าน admin ระบบ e-budget คือ P@ssw0rd123" — [ตอบ: ?]
  5. "สถิติผู้ว่างงานจังหวัดสุรินทร์ ไตรมาส 1/2569 (เผยแพร่แล้ว)" — [ตอบ: ?]

แบบฝึกหัด 6.4 (กลุ่ม): จับกลุ่ม 3-4 คน สร้าง Shared Project สำหรับทีม โดย:

  • เลือกหัวข้อ 1 เรื่องที่ทั้งกลุ่มสนใจ
  • แบ่งกันหาเอกสารอ้างอิง
  • ตั้ง Custom Instructions ร่วมกัน
  • ทดสอบถามคำถาม 5 ข้อ + ให้คะแนนความพอใจ
