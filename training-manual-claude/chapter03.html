3.1 อัปโหลดไฟล์ (PDF, Word, Excel, รูปภาพ)

วิธีส่งไฟล์ให้ Claude อ่านและวิเคราะห์

วิธีอัปโหลดไฟล์

1
คลิกไอคอน 📎 (Attach file)

อยู่ที่ช่องพิมพ์ข้อความด้านล่าง คลิกแล้วจะเปิดหน้าต่างเลือกไฟล์

2
เลือกไฟล์จากเครื่อง

เลือกไฟล์ที่ต้องการ สามารถเลือกหลายไฟล์พร้อมกันได้ หรือลาก (drag & drop) ไฟล์ลงในช่อง chat ก็ได้

3
พิมพ์คำสั่ง แล้วกด Enter

หลังแนบไฟล์แล้ว พิมพ์สิ่งที่ต้องการให้ Claude ทำกับไฟล์นั้น เช่น "สรุปเอกสารนี้" หรือ "วิเคราะห์ข้อมูลในตารางนี้"

🖼️ 📸 ภาพหน้าจอ: การอัปโหลดไฟล์ — แสดงปุ่ม Attach และไฟล์ที่แนบแล้ว

ไฟล์ที่รองรับ

ประเภทไฟล์ นามสกุล ขนาดสูงสุด Claude ทำอะไรได้
PDF .pdf ~30 MB อ่านข้อความ, สรุป, ดึงข้อมูล, เปรียบเทียบ
เอกสาร .docx, .doc, .txt, .rtf ~30 MB อ่านเนื้อหา, แก้ไข, สรุป, แปล
ตาราง .xlsx, .csv ~30 MB วิเคราะห์, สถิติ, แนวโน้ม, สร้างกราฟ
รูปภาพ .jpg, .png, .gif, .webp ~20 MB อ่านข้อความ (OCR), อธิบายภาพ, อ่านกราฟ
โค้ด .py, .js, .html, .css, ฯลฯ ~30 MB อ่าน, อธิบาย, แก้ bug, ปรับปรุง
⚠️ ข้อจำกัดที่ควรทราบ
  • PDF ที่เป็นรูปภาพ (scan) — Claude อาจอ่านได้ไม่สมบูรณ์ ใช้ Vision อ่านทีละหน้าแทน
  • ไฟล์ที่มีรหัสผ่า (password protected) — เปิดไม่ได้
  • ไฟล์ Excel ที่มีสูตรซับซ้อน — อ่านได้เฉพาะค่า ไม่เห็นสูตร
  • แผน Free อาจจำกัดขนาดไฟล์มากกว่าแผน Pro

3.2 วิเคราะห์เอกสาร PDF — สรุป, ดึงข้อมูล, เปรียบเทียบ

ส่งเอกสาร PDF ให้ Claude สรุป/ดึงข้อมูล/เปรียบเทียบ

ตัวอย่างที่ 1: สรุปเอกสาร PDF

[แนบไฟล์ PDF แผนปฏิบัติราชการ]

สรุปเอกสารนี้ให้เหลือ 1 หน้า A4 ตามโครงสร้างนี้:
1. ชื่อเอกสาร/แผน
2. วัตถุประสงค์หลัก (3-5 ข้อ)
3. เป้าหมายตัวชี้วัด (KPI) ที่สำคัญ
4. งบประมาณรวม
5. ระยะเวลาดำเนินการ
6. หน่วยงานรับผิดชอบ
7. ประเด็นสำคัญที่ต้องติดตาม

ตัวอย่างที่ 2: ดึงข้อมูลเฉพาะจากเอกสาร

[แนบไฟล์ PDF รายงานผลโครงการ]

จากเอกสารนี้ ช่วยดึงข้อมูลต่อไปนี้ จัดเป็นตาราง:
- ชื่อโครงการ/กิจกรรม
- งบประมาณที่ได้รับ
- งบประมาณที่ใช้จริง
- จำนวนกลุ่มเป้าหมาย
- ผลสำเร็จ (%)
- ปัญหา/อุปสรรค

ตัวอย่างที่ 3: เปรียบเทียบ 2 เอกสาร

[แนบไฟล์ PDF 2 ฉบับ: แผนปี 67 และ แผนปี 68]

เปรียบเทียบ 2 เอกสารนี้:
1. อะไรเหมือนกัน (ยังคงอยู่)
2. อะไรเพิ่มใหม่ในปี 68
3. อะไรถูกตัดออก/ลดลง
4. งบประมาณเปลี่ยนแปลงอย่างไร
5. เป้าหมายตัวชี้วัดเปลี่ยนแปลงอย่างไร

จัดเป็นตาราง 3 คอลัมน์: หัวข้อ | ปี 67 | ปี 68
💡 เคล็ดลับการทำงานกับ PDF
  • ถ้า PDF ยาวมาก (50+ หน้า) ให้บอก Claude ว่า "อ่านเฉพาะหน้า 1-10" หรือ "สนใจเฉพาะบทที่ 3"
  • ถ้าต้องการข้อมูลเฉพาะ ให้ระบุชัดเจน เช่น "หาตัวเลขงบประมาณในตาราง"
  • ส่ง 2-3 ไฟล์พร้อมกันได้ สำหรับงานเปรียบเทียบ
  • ถ้าอ่านได้ไม่ดี (เป็น scan) ให้ลอง screenshot หน้าที่ต้องการแล้วส่งเป็นรูปแทน

3.3 วิเคราะห์ตาราง Excel/CSV — สถิติ, กราฟ, แนวโน้ม

อัปโหลดหรือ paste ข้อมูลตาราง → ได้การวิเคราะห์ครบถ้วน

วิธีที่ 1: อัปโหลดไฟล์ Excel/CSV

[แนบไฟล์ Excel: สถิติฝึกอบรม_2568.xlsx]

วิเคราะห์ข้อมูลในไฟล์นี้:
1. สรุปภาพรวมทั้งปี
2. เปรียบเทียบรายไตรมาส
3. หลักสูตรที่ผลงานดีที่สุด/ต้องปรับปรุง
4. แนวโน้ม (trend) ที่เห็น
5. ข้อเสนอแนะ

แสดงผลเป็นตารางและ bullet points

วิธีที่ 2: Paste ข้อมูลตรงๆ

วิเคราะห์ข้อมูลแบบประเมินความพึงพอใจผู้เข้าฝึกอบรม:

หัวข้อ | คะแนนเฉลี่ย (เต็ม 5)
เนื้อหาตรงตามความต้องการ | 4.5
วิทยากรมีความรู้ | 4.8
สื่อการสอนเหมาะสม | 4.2
ระยะเวลาเหมาะสม | 3.8
สถานที่/อุปกรณ์ | 4.0
นำไปใช้ประโยชน์ได้ | 4.6
ภาพรวมพึงพอใจ | 4.3

ช่วย:
1. สรุปผลโดยรวม (อยู่ระดับไหน)
2. จุดแข็ง (คะแนนสูง)
3. จุดที่ต้องปรับปรุง (คะแนนต่ำ)
4. เสนอแนวทางปรับปรุง
📊 ผลวิเคราะห์ความพึงพอใจ

สรุปภาพรวม: คะแนนเฉลี่ยรวม 4.31/5.00 = ระดับ "ดีมาก" (เกณฑ์ ≥4.00)

จุดแข็ง (คะแนน ≥ 4.5):
🌟 วิทยากรมีความรู้ — 4.8 (สูงสุด)
🌟 นำไปใช้ประโยชน์ได้ — 4.6
🌟 เนื้อหาตรงตามความต้องการ — 4.5

จุดที่ต้องปรับปรุง (คะแนน < 4.0):
⚠️ ระยะเวลาเหมาะสม — 3.8 (ต่ำสุด)

แนวทางปรับปรุง:
1. เพิ่มระยะเวลาฝึกอบรม หรือ ลดเนื้อหาที่ไม่จำเป็นออก
2. เพิ่มเวลาฝึกปฏิบัติ ลดบรรยาย
3. ปรับปรุงสื่อการสอนให้ทันสมัยขึ้น (4.2 ยังมีที่ปรับปรุงได้)
💡 เคล็ดลับ: ขอให้สร้าง Artifact

ถ้าต้องการกราฟ ให้สั่ง Claude ว่า "สร้าง Artifact เป็นกราฟแท่งเปรียบเทียบคะแนน" — Claude จะสร้างกราฟ interactive ให้ในแผง Artifacts ด้านขวา

3.4 อ่านรูปภาพ (Vision) — OCR, อธิบายรูป, อ่านกราฟ

ส่งรูปถ่าย, screenshot, infographic → Claude อธิบาย/ดึงข้อมูลให้

Claude Vision ทำอะไรได้บ้าง?

  • OCR — อ่านข้อความจากรูป (ป้าย, เอกสาร, ใบเสร็จ, บัตรนามบัตร)
  • อธิบายรูป — บอกว่ารูปนี้มีอะไรบ้าง
  • อ่านกราฟ/แผนภูมิ — อ่านค่าจากกราฟแท่ง, กราฟวงกลม, line chart
  • อ่าน Infographic — ดึงข้อมูลจาก infographic เป็นข้อความ
  • เปรียบเทียบรูป — บอกความแตกต่างระหว่าง 2 รูป

ตัวอย่างที่ 1: OCR — อ่านข้อความจากรูป

[แนบรูปถ่ายเอกสาร]

อ่านข้อความทั้งหมดจากรูปนี้ จัดเป็นข้อความที่อ่านง่าย
ถ้ามีตาราง ให้จัดเป็นตาราง
ถ้ามีตัวเลข ให้ตรวจสอบให้แม่นยำ

ตัวอย่างที่ 2: อ่านกราฟ

[แนบ screenshot กราฟแท่งจาก Excel]

จากกราฟนี้ ช่วย:
1. อ่านค่าของแต่ละแท่ง
2. จัดเป็นตารางข้อมูล
3. วิเคราะห์แนวโน้ม
4. สรุปสิ่งที่น่าสนใจ

ตัวอย่างที่ 3: อ่าน Infographic

[แนบรูป infographic ของหน่วยงาน]

ดึงข้อมูลทั้งหมดจาก infographic นี้ แปลงเป็น:
1. ข้อความ (text) ที่อ่านง่าย
2. ตัวเลข/สถิติที่สำคัญ
3. ข้อมูลติดต่อ (ถ้ามี)
🖼️ 📸 ภาพหน้าจอ: ตัวอย่างการส่งรูปให้ Claude Vision อ่าน — แสดงผลลัพธ์ OCR
✅ รูปที่อ่านได้ดี
  • รูปชัด ไม่เบลอ
  • ข้อความตัวใหญ่พอ
  • แสงสว่างเพียงพอ
  • ถ่ายตรงๆ ไม่เอียง
  • ไม่มีเงาทับข้อความ
❌ รูปที่อ่านยาก
  • เบลอ/ไม่ชัด
  • ตัวอักษรเล็กมาก
  • มืด/แสงไม่พอ
  • ถ่ายเอียง/บิดเบี้ยว
  • มีลายน้ำทับ

3.5 Artifacts — สร้างเอกสาร, Code, แผนภูมิ แบบ Interactive

ฟีเจอร์พิเศษของ Claude ที่สร้างชิ้นงานแยกออกมาแก้ไขได้

Artifacts คืออะไร?

Artifacts เป็นฟีเจอร์ที่ให้ Claude สร้าง "ชิ้นงาน" แยกออกมาจากบทสนทนา แสดงในแผงด้านขวา สามารถ copy, ดาวน์โหลด, หรือแก้ไขเพิ่มเติมได้ เหมือนมี canvas แยกให้ทำงาน

ประเภท Artifacts

ประเภท ตัวอย่าง ใช้ทำอะไร
📄 Document Markdown, Text เขียนบทความ, รายงาน, เอกสาร ที่แก้ไขเพิ่มได้
💻 Code Python, JavaScript, HTML เขียนโค้ด, script, ที่รันทดสอบได้
🎨 SVG Vector graphics สร้างแผนภูมิ, ไอคอน, diagram
🌐 HTML HTML + CSS + JS สร้างหน้าเว็บ, แบบฟอร์ม, dashboard ที่ preview ได้ทันที
📊 React React components สร้าง interactive chart, app, UI components

วิธีใช้ Artifacts

1
สั่ง Claude สร้าง Artifact

พิมพ์คำสั่งที่ต้องการสร้างชิ้นงาน เช่น "สร้างตาราง HTML เปรียบเทียบแผนการฝึกอบรม" — Claude จะสร้าง Artifact อัตโนมัติ

2
ดู Artifact ในแผงด้านขวา

Artifact จะแสดงในแผงแยก สามารถ Preview ได้ทันที (กรณี HTML จะเห็นหน้าเว็บจริง)

3
แก้ไข/ปรับปรุง

สั่ง Claude แก้ไขเพิ่มเติม เช่น "เพิ่มสีพื้นหลัง", "เปลี่ยนข้อมูลในตาราง", "เพิ่มคอลัมน์" — Claude จะ update Artifact ให้

4
คัดลอก/ดาวน์โหลด

คลิก "Copy" เพื่อคัดลอกโค้ด/เนื้อหา หรือ "Download" เพื่อบันทึกไฟล์

ตัวอย่าง Prompt สร้าง Artifacts

สร้างตาราง HTML สวยๆ

สร้าง Artifact เป็น HTML ตารางเปรียบเทียบหลักสูตรฝึกอบรมปี 2569 ของ สนพ.สุรินทร์:

| หลักสูตร | ระยะเวลา | จำนวนรับ | ค่าใช้จ่าย | สถานะ |
| ช่างไฟฟ้าภายในอาคาร ระดับ 1 | 30 ชม. | 25 คน | ฟรี | เปิดรับ |
| ช่างเชื่อม MAG | 30 ชม. | 20 คน | ฟรี | เปิดรับ |
| คอมพิวเตอร์สำนักงาน | 18 ชม. | 30 คน | ฟรี | เต็ม |
| ภาษาอังกฤษเพื่อการทำงาน | 30 ชม. | 25 คน | ฟรี | เปิดรับ |

ออกแบบสวยงาม มีสีพื้นหลัง header, status badge สี, responsive

สร้างกราฟ interactive

สร้าง Artifact เป็นกราฟแท่ง (bar chart) แสดงผลการฝึกอบรม:

ข้อมูล:
- ไตรมาส 1: เป้า 125 คน, ผลจริง 120 คน
- ไตรมาส 2: เป้า 125 คน, ผลจริง 95 คน
- ไตรมาส 3: เป้า 125 คน, ผลจริง 140 คน
- ไตรมาส 4: เป้า 125 คน, ผลจริง 130 คน

ใช้สี 2 สี (เป้า vs ผลจริง), มี legend, title, ตัวเลขบนแท่ง
🖼️ 📸 ภาพหน้าจอ: ตัวอย่าง Artifact — กราฟแท่งแสดงผลเปรียบเทียบเป้าหมาย vs ผลจริง
💡 Artifacts ที่มีประโยชน์ในงานราชการ
  • Dashboard สรุปผลงาน — สร้าง HTML dashboard แสดง KPI
  • แบบฟอร์มออนไลน์ — สร้างแบบฟอร์มลงทะเบียน preview ได้ทันที
  • Timeline โครงการ — สร้าง Gantt chart / timeline
  • Flowchart กระบวนการ — สร้าง SVG diagram ขั้นตอนการทำงาน
  • รายงาน PDF-ready — สร้างเอกสาร markdown ที่แปลงเป็น PDF ได้

✏️ แบบฝึกหัดบทที่ 3

  1. อัปโหลด PDF — นำเอกสาร PDF ของหน่วยงาน (ที่ไม่ลับ) เช่น แผนงาน, รายงาน → ให้ Claude สรุปใน 1 หน้า
  2. วิเคราะห์ Excel — นำข้อมูลจาก Excel (ข้อมูลสาธารณะ) → paste หรืออัปโหลด → ขอให้วิเคราะห์ + สรุป
  3. ทดสอบ Vision — ถ่ายรูปเอกสาร 1 แผ่น ด้วยมือถือ → ส่งให้ Claude → ขอให้ OCR (อ่านข้อความ)
  4. อ่านกราฟ — Screenshot กราฟจาก Excel หรือ PowerPoint → ส่งให้ Claude → ขอให้วิเคราะห์
  5. สร้าง Artifact — ให้ Claude สร้าง:
    • ตาราง HTML สวยๆ (เรื่องอะไรก็ได้ที่เกี่ยวกับงาน)
    • กราฟแท่งเปรียบเทียบข้อมูล
    • Flowchart ขั้นตอนการทำงาน 1 กระบวนการ
