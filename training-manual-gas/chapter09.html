บทที่ 9: Deploy Web App

เผยแพร่ Web App ให้คนอื่นใช้งานได้ — ขั้นตอน Deploy, อัปเดต, แก้ปัญหา

🎯 วัตถุประสงค์การเรียนรู้

  • Deploy Web App ได้อย่างถูกต้อง
  • เข้าใจเรื่อง Permission (สิทธิ์การเข้าถึง)
  • อัปเดต Web App เมื่อแก้ไขโค้ด
  • แก้ปัญหาที่พบบ่อยในการ Deploy

📚 Deploy คืออะไร?

Deploy คือการ "เผยแพร่" Script ให้คนอื่นเข้าถึงได้ผ่าน URL เปรียบเหมือนการเปิดร้านค้า — เขียนโค้ดเสร็จ (สร้างร้าน) แต่ถ้าไม่ Deploy (เปิดร้าน) คนอื่นก็ใช้ไม่ได้

🔄 ก่อน vs หลัง Deploy

ก่อน Deploy มีแค่เราใช้ได้ — รันจาก Script Editor เท่านั้น
หลัง Deploy ได้ URL — ใครก็เปิดใช้งานได้ (ตามสิทธิ์ที่กำหนด)

🚀 ขั้นตอนการ Deploy

1

เปิด Script Editor → คลิก "Deploy" (มุมขวาบน)

2

เลือก "New deployment"

3

คลิก ⚙️ → เลือก "Web app"

4

ตั้งค่า:

⚙️ ตั้งค่า Deployment

Description คำอธิบาย เช่น "v1.0 - ระบบลงทะเบียน"
Execute as Me — ทำงานด้วยสิทธิ์ของเรา (แนะนำ)
User accessing — ทำงานด้วยสิทธิ์ของผู้เข้าใช้
Who has access Anyone — ทุกคนเข้าถึงได้ (แนะนำสำหรับ Web App สาธารณะ)
Anyone with Google account — ต้อง login Google ก่อน
Only myself — เฉพาะเรา
5

คลิก "Deploy"

6

ครั้งแรกจะขอ Authorization — กด "Authorize access" → เลือกบัญชี → "Allow"

7

ได้ URL สำหรับเข้าใช้งาน → คัดลอกเก็บไว้!

ตัวอย่าง URL:
https://script.google.com/macros/s/AKfycbz9C1KOVt13HRy5aO.../exec

🔄 อัปเดต Web App (หลังแก้ไขโค้ด)

สำคัญมาก! เมื่อแก้ไขโค้ดแล้ว ต้อง Deploy ใหม่ถึงจะเห็นการเปลี่ยนแปลง

วิธีที่ 1: Deploy ใหม่ (URL ไม่เปลี่ยน)

1

Deploy → "Manage deployments"

2

คลิก ✏️ (Edit) ที่ deployment ที่ต้องการ

3

ที่ Version เลือก "New version"

4

ใส่ Description (เช่น "v2.0 - เพิ่มฟีเจอร์ X") → Deploy

วิธีที่ 2: ใช้ Test URL (สำหรับทดสอบ)

// Test URL ใช้โค้ดล่าสุดเสมอ (ไม่ต้อง deploy ใหม่)
// แต่ใช้ได้เฉพาะตัวเอง

// Deploy → Test deployments → Copy URL
// URL จะลงท้ายด้วย /dev แทน /exec
⚠️ ความแตกต่างระหว่าง /exec กับ /dev
/exec URL สาธารณะ — ใช้ version ที่ deploy ล่าสุด — ทุกคนเข้าถึงได้
/dev URL ทดสอบ — ใช้โค้ดปัจจุบัน — เฉพาะเจ้าของเข้าถึงได้

🔐 Permission (สิทธิ์)

// Execute as: Me
// → GAS ใช้สิทธิ์ของเรา (เจ้าของ Script) ในการทำงาน
// → ผู้ใช้ไม่ต้อง login Google
// → ผู้ใช้ไม่ต้องมีสิทธิ์เข้าถึง Sheet
// → เหมาะกับ Web App สาธารณะ

// Execute as: User accessing the web app
// → GAS ใช้สิทธิ์ของผู้เข้าใช้
// → ผู้ใช้ต้อง login Google
// → ผู้ใช้ต้องมีสิทธิ์เข้าถึง Sheet ที่เกี่ยวข้อง
// → เหมาะกับ Web App ภายในองค์กร

🐛 ปัญหาที่พบบ่อยและวิธีแก้

❌ ปัญหาที่ 1: "This app isn't verified"

สาเหตุ: Google ยังไม่ verify app ของเรา (ปกติสำหรับ app ภายใน)

วิธีแก้: คลิก "Advanced" → "Go to [project name] (unsafe)" → "Allow"

❌ ปัญหาที่ 2: แก้โค้ดแล้วไม่เห็นการเปลี่ยนแปลง

สาเหตุ: ยังไม่ได้ Deploy version ใหม่

วิธีแก้: Manage deployments → Edit → New version → Deploy

❌ ปัญหาที่ 3: "Authorization required"

สาเหตุ: เพิ่ม Service ใหม่ (เช่น DriveApp) แต่ยังไม่ได้ขอสิทธิ์

วิธีแก้: รันฟังก์ชันจาก Editor → Authorize → Deploy ใหม่

❌ ปัญหาที่ 4: "Script function not found: doGet"

สาเหตุ: ไม่มีฟังก์ชัน doGet() ใน Code.gs

วิธีแก้: ตรวจสอบว่ามี function doGet(e) { ... } อยู่ในไฟล์

📋 Deploy Checklist

// ✅ Checklist ก่อน Deploy
// 1. ทดสอบฟังก์ชันทุกตัวจาก Editor แล้ว ✓
// 2. ตรวจ Error ใน Execution log แล้ว ✓
// 3. ทดสอบจาก /dev URL แล้ว ✓
// 4. ใส่ try-catch ป้องกัน crash แล้ว ✓
// 5. ตรวจ Permission (Execute as / Who has access) ✓
// 6. ใส่ Description ที่สื่อความหมาย ✓
// 7. จดบันทึก Version + วันที่ Deploy ✓

🔬 Lab 11: Deploy Web App แรก

เวลา: 20 นาที

โจทย์: Deploy Web App ง่ายๆ แล้วแชร์ URL ให้เพื่อน

1

สร้าง Code.gs:

function doGet(e) {
  return HtmlService.createHtmlOutputFromFile("index")
    .setTitle("Web App แรกของฉัน");
}

function getServerTime() {
  return Utilities.formatDate(new Date(), "Asia/Bangkok", "dd/MM/yyyy HH:mm:ss");
}

function getMyName() {
  return "ชื่อของฉัน";  // ← เปลี่ยนเป็นชื่อจริง
}
2

สร้าง index.html:

<!DOCTYPE html>
<html>
<head>
  <base target="_top">
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
  <div class="container text-center mt-5">
    <h1>🎉 Web App แรกของฉัน!</h1>
    <p class="lead">Deploy สำเร็จแล้ว!</p>
    <div class="card mx-auto" style="max-width:400px">
      <div class="card-body">
        <p>ชื่อ: <strong id="name">กำลังโหลด...</strong></p>
        <p>เวลา Server: <strong id="time">กำลังโหลด...</strong></p>
        <button onclick="refresh()" class="btn btn-primary">🔄 Refresh</button>
      </div>
    </div>
  </div>
  <script>
    window.onload = function() { refresh(); };
    function refresh() {
      google.script.run.withSuccessHandler(function(t) {
        document.getElementById('time').textContent = t;
      }).getServerTime();
      google.script.run.withSuccessHandler(function(n) {
        document.getElementById('name').textContent = n;
      }).getMyName();
    }
  </script>
</body>
</html>
3

Deploy → Web app → Execute as: Me → Anyone → Deploy

4

คัดลอก URL แล้วเปิดในเบราว์เซอร์ — ควรเห็นหน้าเว็บ!

5

แชร์ URL ให้เพื่อนทดสอบ

💡 Tips

  • ทดสอบด้วย /dev ก่อน: ไม่ต้อง deploy ทุกครั้งที่แก้โค้ด
  • Version: ใส่ Description ที่สื่อความหมาย เช่น "v3 - เพิ่มปุ่มลบ"
  • URL ไม่เปลี่ยน: เมื่อ update deployment URL เดิมยังใช้ได้
  • Deploy ID: ส่วนที่อยู่ระหว่าง /s/ กับ /exec คือ Deploy ID — เปลี่ยนเมื่อสร้าง New deployment
  • จดบันทึก: บันทึก Version, วันที่, สิ่งที่เปลี่ยน ทุกครั้งที่ deploy

📝 สรุปบทที่ 9

Deploy เผยแพร่ Script เป็น URL ที่คนอื่นเข้าถึงได้
New deployment สร้าง URL ใหม่ (Deploy ID ใหม่)
Update deployment อัปเดตโค้ด โดย URL เดิมยังใช้ได้
/exec vs /dev exec = สาธารณะ (version ล่าสุด), dev = ทดสอบ (โค้ดปัจจุบัน)
Permission Execute as Me + Anyone = Web App สาธารณะ
