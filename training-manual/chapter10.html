10.1 Kiro Spec Mode — สร้างระบบแบบมีแบบแปลน

Spec Mode คือโหมดพิเศษของ Kiro ที่ช่วยวางแผนก่อนเขียนโค้ด — เหมือนมีสถาปนิกช่วยวาดแบบก่อนสร้างบ้าน:

ขั้นตอนใช้ Spec Mode
  1. เริ่ม Spec — กด "Start Spec" ใน Kiro หรือพิมพ์ "สร้าง spec สำหรับ..."
  2. Requirements — Kiro ถามว่าระบบต้องทำอะไรได้บ้าง
  3. Design — Kiro ออกแบบโครงสร้าง (ไฟล์, ฟังก์ชัน, Sheets)
  4. Tasks — Kiro แบ่งงานเป็นขั้นตอนย่อย
  5. Execute — Kiro สร้างโค้ดทีละ Task
ตัวอย่าง: สร้างระบบรับเรื่องร้องเรียนด้วย Spec
📝 Prompt เริ่มต้น
สร้าง spec สำหรับระบบรับเรื่องร้องเรียนออนไลน์

ความต้องการ:
- ประชาชนแจ้งเรื่องร้องเรียนผ่านฟอร์ม (ชื่อ, เบอร์โทร, เรื่อง, รายละเอียด, แนบรูป)
- ระบบออกเลขรับเรื่อง (COM-001)
- ส่ง Email + Line แจ้งเตือน Admin
- ประชาชนตรวจสอบสถานะด้วยเลขรับเรื่อง
- Admin จัดการ: รับเรื่อง → กำลังดำเนินการ → เสร็จสิ้น
- พิมพ์รายงานสรุปรายเดือน

เทคโนโลยี: GAS + Sheets + Bootstrap 5
ข้อจำกัด: ห้ามใช้ template literal, ห้ามใช้ SVG
Kiro จะสร้างให้:
📋 Requirements
  • FR-1: ผู้ใช้ส่งเรื่องร้องเรียน
  • FR-2: ระบบออกเลขรับเรื่อง
  • FR-3: แจ้งเตือนอัตโนมัติ
  • FR-4: ตรวจสอบสถานะ
  • FR-5: Admin จัดการเรื่อง
  • FR-6: พิมพ์รายงาน
🏗️ Design
  • Code.gs (Backend)
  • index.html (ฟอร์ม)
  • admin.html (จัดการ)
  • check.html (ตรวจสถานะ)
  • Sheet: COMPLAINTS
  • Sheet: SETTINGS
✅ Tasks
  • Task 1: สร้าง Sheet + headers
  • Task 2: Code.gs ฟังก์ชันหลัก
  • Task 3: index.html ฟอร์ม
  • Task 4: admin.html จัดการ
  • Task 5: Email + Line
  • Task 6: Print page

10.2 เทคนิคขั้นสูง — Sub-agents, Custom Agents

Sub-agents คืออะไร?

Sub-agents คือ "ผู้ช่วย" ที่ Kiro แยกไปทำงานเฉพาะทาง — เหมือนมอบหมายงานให้ลูกน้อง:

🤖 Sub-agent ทำอะไร
  • วิเคราะห์โค้ดเดิมก่อนแก้ไข
  • ค้นหาไฟล์ที่เกี่ยวข้อง
  • ทำงาน Task ย่อยแล้วรายงานกลับ
  • สร้างไฟล์หลายไฟล์พร้อมกัน
📌 ใช้เมื่อไร
  • งานซับซ้อนที่ต้องทำหลายขั้นตอน
  • ต้องวิเคราะห์โค้ดเดิมก่อนแก้
  • สร้างระบบใหม่จากศูนย์
  • Refactor โค้ดขนาดใหญ่
MCP Connections

MCP (Model Context Protocol) ช่วยให้ Kiro เชื่อมต่อกับเครื่องมือภายนอก:

MCP Serverใช้ทำอะไร
File Systemอ่าน/เขียนไฟล์บนเครื่อง
Gitจัดการ version control
Databaseเชื่อมต่อฐานข้อมูล
Web Searchค้นหาข้อมูลจากอินเทอร์เน็ต
Custom APIเชื่อมต่อ API ภายนอก
💡 สำหรับผู้เรียนขั้นสูง: ลองสร้าง Custom Agent ที่เชี่ยวชาญเรื่อง GAS โดยเฉพาะ — ใส่ความรู้เรื่อง Sheets API, quotas, best practices เข้าไป

10.3 Workshop: ผู้เข้าอบรมเลือกเพิ่มฟีเจอร์

🛠️ Workshop! เลือกฟีเจอร์ที่ต้องการเพิ่มให้ระบบลงทะเบียน (อย่างน้อย 1 ข้อ)
ฟีเจอร์ที่แนะนำ (เลือกตามความสนใจ):
⭐ ง่าย (30 นาที)
  1. เพิ่มสีสถานะ (Badge) ในตาราง
  2. เพิ่ม Export CSV
  3. เพิ่ม Confirm dialog ก่อนลบ
  4. เพิ่มนับจำนวนที่นั่งคงเหลือ
  5. เพิ่ม Dark Mode toggle
⭐⭐ ปานกลาง (1 ชม.)
  1. ระบบค้นหา + กรองสถานะ
  2. Email แจ้งเตือนเมื่อลงทะเบียน
  3. หน้าพิมพ์รายชื่อ (A4)
  4. เพิ่มอัปโหลดรูปถ่าย
  5. Dashboard สถิติ (Chart)
⭐⭐⭐ ยาก (2+ ชม.)
  1. ระบบ Login + Admin/User
  2. Line Notify แจ้งเตือน
  3. QR Code สำหรับเช็คชื่อ
  4. ระบบประเมินผลหลังอบรม
  5. PDF Certificate อัตโนมัติ
🔥 ท้าทาย (ครึ่งวัน)
  1. ระบบจัดการหลักสูตร (CRUD)
  2. Calendar แสดงตารางอบรม
  3. รายงานสรุปรายเดือน + กราฟ
  4. Webhook รับข้อมูลจากภายนอก
  5. ระบบ Backup อัตโนมัติ
📝 วิธีทำ: เลือกฟีเจอร์ → เขียน Prompt ให้ Kiro → ทดสอบ → แก้ Bug → Deploy!

10.4 นำเสนอผลงาน

รูปแบบการนำเสนอ (5 นาที/ทีม)
  1. แนะนำระบบ (30 วินาที) — ชื่อระบบ, แก้ปัญหาอะไร
  2. สาธิตการใช้งาน (2 นาที) — โชว์หน้าจอจริง
  3. อธิบายเทคนิค (1 นาที) — ใช้ Kiro อย่างไร, Prompt อะไร
  4. ปัญหาที่พบ + วิธีแก้ (1 นาที)
  5. แผนต่อยอด (30 วินาที) — จะเพิ่มอะไรต่อ
เกณฑ์การประเมิน
เกณฑ์คะแนนรายละเอียด
🎯 ฟังก์ชันทำงานได้30%ระบบทำงานได้จริง ไม่มี error
🎨 UI/UX20%สวยงาม ใช้ง่าย responsive
💡 ความคิดสร้างสรรค์20%ฟีเจอร์แปลกใหม่ แก้ปัญหาจริง
🤖 การใช้ Kiro15%Prompt ดี ใช้ Kiro ได้มีประสิทธิภาพ
📢 การนำเสนอ15%ชัดเจน ตรงเวลา ตอบคำถามได้

10.5 Tips & Troubleshooting — ปัญหาที่พบบ่อย

🔧 Top 10 ปัญหาและวิธีแก้

สาเหตุ: Script ID ใน .clasp.json ผิด
แก้ไข: ตรวจ .clasp.json → ใส่ Script ID ที่ถูกต้อง (ดูจาก URL ของ Script Editor)

สาเหตุ: Deploy ใหม่แต่ใช้ URL เก่า หรือยังไม่ deploy
แก้ไข: clasp deploy ใหม่ → ใช้ URL ใหม่ หรือตั้ง Execute as: Me, Access: Anyone

สาเหตุ: เปิดไฟล์ HTML ใน browser โดยตรง (ไม่ผ่าน GAS deploy)
แก้ไข: ต้องเปิดผ่าน Deploy URL หรือ Test Deployment เท่านั้น

สาเหตุ: ชื่อ Sheet ไม่ตรง, column ผิดลำดับ, หรือ permission
แก้ไข: ตรวจ getSheetByName('ชื่อ') ตรงกับชื่อ tab, ตรวจ appendRow() ลำดับคอลัมน์

สาเหตุ: GAS HTML Sanitizer ลบ SVG attributes (viewBox, path)
แก้ไข: ใช้ Emoji หรือ Bootstrap Icons CDN แทน — ห้ามใช้ inline SVG

สาเหตุ: GAS HTML Service ตีความ backtick ผิดในบาง context
แก้ไข: ใช้ string concatenation ('abc' + variable + 'def') แทน

สาเหตุ: Browser cache หรือใช้ URL ของ deploy เก่า
แก้ไข: Ctrl+Shift+R (hard refresh), ตรวจว่าใช้ Deploy URL ล่าสุด

สาเหตุ: เกิน quota (100/วัน free) หรือยังไม่ authorize
แก้ไข: ตรวจ quota ที่ script.google.com > Settings > Quotas, run ฟังก์ชันใน Editor เพื่อ authorize

สาเหตุ: มีคนอื่น push ก่อน หรือลืม pull
แก้ไข: git pull origin main → แก้ conflict (ถ้ามี) → git push อีกครั้ง

สาเหตุ: ไม่ได้ระบุเทคโนโลยีใน Prompt หรือไม่มี Steering
แก้ไข: ระบุ "Google Apps Script" ใน Prompt + ตั้งค่า Steering file
