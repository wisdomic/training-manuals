7.1 เทคนิคเขียน Prompt ที่ดี

Prompt ที่ดีจะทำให้ Kiro สร้างโค้ดได้ตรงตามต้องการ ประหยัดเวลาแก้ไข:

✅ Do — ควรทำ
  1. ชัดเจน — บอกว่าต้องการอะไร ทำอะไร
  2. ระบุเทคโนโลยี — GAS, Sheets, Bootstrap 5
  3. ให้ตัวอย่าง — โครงสร้างข้อมูล, output ที่คาดหวัง
  4. แบ่งขั้นตอน — ทำทีละส่วน ไม่สั่งรวดเดียว
  5. ระบุข้อจำกัด — ห้ามใช้อะไร, ต้องรองรับอะไร
❌ Don't — ไม่ควรทำ
  1. คลุมเครือ — "ทำระบบให้หน่อย"
  2. ไม่ระบุเทคโนโลยี — อาจได้ Node.js แทน GAS
  3. สั่งรวดเดียว — "ทำทุกอย่างเลย" (ผลลัพธ์จะไม่ดี)
  4. ไม่ให้บริบท — ไม่บอกโครงสร้าง Sheet
  5. คาดหวังสมบูรณ์ครั้งเดียว — ต้อง iterate
📝 สูตร Prompt ที่ดี
[บทบาท] + [เทคโนโลยี] + [สิ่งที่ต้องการ] + [รายละเอียด] + [ข้อจำกัด]

ตัวอย่าง:
"สร้างฟังก์ชัน Google Apps Script ชื่อ getTrainees()
ที่อ่านข้อมูลจาก Sheet 'TRAINEES' (Sheet ID: xxx)
return เป็น Array of Objects
คอลัมน์: trainee_id, prefix, first_name, last_name, phone, email
ข้อจำกัด: ห้ามใช้ template literal"
ตัวอย่าง Prompt ที่ดี vs ไม่ดี
❌ ไม่ดี✅ ดี
"ทำหน้าฟอร์มให้หน่อย" "สร้างหน้า HTML ฟอร์มลงทะเบียนด้วย Bootstrap 5 มี field: คำนำหน้า (dropdown), ชื่อ, นามสกุล, เบอร์โทร, เลือกหลักสูตร (dropdown โหลดจาก Backend)"
"ทำ Backend" "สร้าง Code.gs สำหรับระบบลงทะเบียน มี doGet() แสดง index.html, ฟังก์ชัน getCourses() อ่านจาก Sheet COURSES, ฟังก์ชัน registerTrainee(data) เขียนลง Sheet TRAINEES"

Steering คือไฟล์กฎที่บอก Kiro ว่าโปรเจกต์นี้ใช้เทคโนโลยีอะไร มีข้อจำกัดอะไร เพื่อให้ Kiro สร้างโค้ดได้ถูกต้องทุกครั้ง:

ขั้นตอนสร้าง Steering

  2. สร้างไฟล์ project-rules.md
  3. ใส่กฎตามตัวอย่างด้านล่าง
ตัวอย่าง .kiro/steering/project-rules.md
# Project Rules — ระบบลงทะเบียนฝึกอบรม

## เทคโนโลยี
- Google Apps Script (GAS)
- Google Sheets เป็นฐานข้อมูล
- Bootstrap 5 สำหรับ Frontend
- Font: Sarabun (Google Fonts)

## โครงสร้าง Sheets
- COURSES: course_id, course_name, hours, start_date, end_date, max_seats, status
- TRAINEES: trainee_id, prefix, first_name, last_name, id_card, phone, email, created_at
- REGISTRATIONS: reg_id, trainee_id, course_id, reg_date, status

## กฎการเขียนโค้ด
- ห้ามใช้ template literal (backtick) ใน HTML files
- ใช้ var แทน let/const ใน .gs files
- ทุกฟังก์ชันต้องมี try-catch
- return ข้อมูลเป็น JSON format: { success: true/false, data: ..., error: ... }
- ภาษาไทยในตัวแปร UI, ภาษาอังกฤษในชื่อฟังก์ชัน/ตัวแปร

## ข้อจำกัด GAS
- ห้ามใช้ SVG (GAS sanitizer ทำลาย)
- ลิงก์ข้ามหน้าต้องมี target="_top"
- ขนาดข้อมูลผ่าน google.script.run ไม่เกิน 256KB

## Sheet ID
- Spreadsheet ID: YOUR_SHEET_ID_HERE
💡 ประโยชน์ Steering: เมื่อตั้งค่าแล้ว ทุกครั้งที่สั่ง Kiro สร้างโค้ด มันจะปฏิบัติตามกฎเหล่านี้โดยอัตโนมัติ — ไม่ต้องพิมพ์ซ้ำทุกครั้ง!

7.4 Workshop: ให้ Kiro สร้าง Code.gs (Backend)

📝 Prompt ที่ใช้สั่ง Kiro:
Prompt #1 — สร้าง Backend หลัก
สร้างไฟล์ Code.gs สำหรับระบบลงทะเบียนฝึกอบรม ด้วย Google Apps Script

Sheet ID: [ใส่ Sheet ID ของคุณ]

ต้องมีฟังก์ชันดังนี้:
1. doGet(e) — แสดง index.html (หน้าลงทะเบียน) หรือ admin.html (ถ้า ?page=admin)
2. include(filename) — สำหรับ include ไฟล์ CSS/JS
3. getCourses() — อ่านหลักสูตรจาก Sheet COURSES ที่ status = "เปิดรับ"
   return: [{ course_id, course_name, hours, start_date, end_date }]
4. registerTrainee(data) — รับข้อมูลผู้สมัคร + หลักสูตร
   - สร้าง trainee_id ใหม่ (T + running number)
   - เขียนลง TRAINEES
   - สร้าง reg_id ใหม่ (R + running number)
   - เขียนลง REGISTRATIONS (status = "รอยืนยัน")
   - return: { success: true, reg_id: 'R001' }
5. getRegistrations() — อ่านการลงทะเบียนทั้งหมด JOIN กับ TRAINEES + COURSES
6. updateStatus(reg_id, newStatus) — เปลี่ยนสถานะลงทะเบียน
7. deleteRegistration(reg_id) — ลบการลงทะเบียน

กฎ:
- ใช้ var (ไม่ใช้ let/const)
- ทุกฟังก์ชันมี try-catch
- return JSON format: { success: true/false, data/error }
- ห้ามใช้ template literal
✅ ผลลัพธ์ที่คาดหวังจาก Kiro:
// Code.gs — ระบบลงทะเบียนฝึกอบรม

var SHEET_ID = 'YOUR_SHEET_ID';

function doGet(e) {
  try {
    var page = (e && e.parameter && e.parameter.page) || 'index';
    var html = HtmlService.createTemplateFromFile(page);
    return html.evaluate()
      .setTitle('ระบบลงทะเบียนฝึกอบรม')
      .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);
  } catch (err) {
    return HtmlService.createHtmlOutput('Error: ' + err.toString());
  }
}

function include(filename) {
  return HtmlService.createHtmlOutputFromFile(filename).getContent();
}

function getCourses() {
  try {
    var ss = SpreadsheetApp.openById(SHEET_ID);
    var sheet = ss.getSheetByName('COURSES');
    var data = sheet.getDataRange().getValues();
    var headers = data[0];
    var courses = [];
    
    for (var i = 1; i < data.length; i++) {
      if (String(data[i][6]) === 'เปิดรับ') {  // status column
        var obj = {};
        for (var j = 0; j < headers.length; j++) {
          obj[headers[j]] = data[i][j];
        }
        courses.push(obj);
      }
    }
    return { success: true, data: courses };
  } catch (err) {
    return { success: false, error: err.toString() };
  }
}

function registerTrainee(formData) {
  try {
    var ss = SpreadsheetApp.openById(SHEET_ID);
    
    // สร้าง trainee_id
    var tSheet = ss.getSheetByName('TRAINEES');
    var tData = tSheet.getDataRange().getValues();
    var traineeId = 'T' + String(tData.length).padStart(3, '0');
    
    // เขียนลง TRAINEES
    tSheet.appendRow([
      traineeId, formData.prefix, formData.first_name,
      formData.last_name, formData.id_card, formData.phone,
      formData.email, formData.organization, new Date()
    ]);
    
    // สร้าง reg_id + เขียนลง REGISTRATIONS
    var rSheet = ss.getSheetByName('REGISTRATIONS');
    var rData = rSheet.getDataRange().getValues();
    var regId = 'R' + String(rData.length).padStart(3, '0');
    
    rSheet.appendRow([
      regId, traineeId, formData.course_id,
      new Date(), 'รอยืนยัน', ''
    ]);
    
    return { success: true, data: { reg_id: regId, trainee_id: traineeId } };
  } catch (err) {
    return { success: false, error: err.toString() };
  }
}

// ฟังก์ชันเปลี่ยนสถานะ (Frontend)
function changeStatus(regId, newStatus) {
  if (!confirm('ต้องการเปลี่ยนสถานะเป็น "' + newStatus + '"?')) return;
  
  showLoading(true);
  google.script.run
    .withSuccessHandler(function(result) {
      showLoading(false);
      if (result.success) {
        showAlert('success', 'เปลี่ยนสถานะสำเร็จ');
        loadData(); // โหลดข้อมูลใหม่
      } else {
        showAlert('danger', 'เกิดข้อผิดพลาด: ' + result.error);
      }
    })
    .withFailureHandler(function(err) {
      showLoading(false);
      showAlert('danger', err.message);
    })
    .updateStatus(regId, newStatus);
}

// ฟังก์ชันลบ (Frontend)
function deleteReg(regId) {
  if (!confirm('⚠️ ยืนยันลบ ' + regId + '?')) return;
  
  google.script.run
    .withSuccessHandler(function(result) {
      if (result.success) {
        showAlert('success', 'ลบสำเร็จ');
        loadData();
      }
    })
    .deleteRegistration(regId);
}

7.7 ทดสอบ + แก้ Bug ด้วย Kiro

วิธี Debug ใน GAS
วิธีใช้เมื่อไรวิธีทำ
Logger.log() ดูค่าตัวแปรใน Backend View > Logs ใน Script Editor
console.log() ดูค่าตัวแปรใน Frontend F12 > Console ในเบราว์เซอร์
try-catch จับ error ไม่ให้ระบบล่ม ใส่ครอบทุกฟังก์ชัน
alert() แสดงค่าแบบ popup (ชั่วคราว) ใส่ตรงจุดที่สงสัย
📝 ตัวอย่าง Prompt แก้ Bug
Bug: Dropdown หลักสูตรไม่แสดงข้อมูล
ช่วยแก้ bug:
- ปัญหา: dropdown เลือกหลักสูตร ไม่แสดงรายการ (ว่างเปล่า)
- สิ่งที่ตรวจแล้ว: 
  - Sheet COURSES มีข้อมูล 3 หลักสูตร status = "เปิดรับ"
  - Console ไม่มี error
  - getCourses() เมื่อ run ใน Script Editor ได้ข้อมูลถูกต้อง
- คาดว่า: อาจเป็นปัญหา column index ใน getCourses() หรือ element id ไม่ตรงกัน

กรุณาตรวจสอบ:
