3.1 หน้าจอ Kiro — สำรวจ Interface

Kiro สร้างบนพื้นฐาน VS Code ดังนั้นผู้ที่เคยใช้ VS Code จะคุ้นเคย แต่มีเพิ่ม AI Panel ด้านขวา

ส่วนประกอบหลัก 5 ส่วน:
1. Activity Bar (ซ้ายสุด)ไอคอน Explorer, Search, Git, Extensions, Kiro Features
2. Side Panel (ซ้าย)แสดงไฟล์, ผลการค้นหา, Git changes
3. Editor (กลาง)พื้นที่แก้ไขโค้ด — เปิดหลายไฟล์เป็น tab ได้
4. Kiro Chat (ขวา)แชทกับ AI — สั่งงาน, ถาม, สร้างโค้ด
5. Terminal (ล่าง)รันคำสั่ง command line (git, clasp, npm)
📸 ภาพหน้าจอ: Kiro IDE พร้อมอธิบายแต่ละส่วน
Keyboard Shortcuts ที่ควรรู้
Shortcutการทำงาน
Ctrl + Shift + PCommand Palette (ค้นหาคำสั่ง)
Ctrl + `เปิด/ปิด Terminal
Ctrl + Bเปิด/ปิด Side Panel
Ctrl + Sบันทึกไฟล์
Ctrl + Shift + Eเปิด Explorer

3.2 Vibe Mode — สนทนาสร้างโค้ด

Vibe Mode คือโหมดแชทอิสระ — คุณพิมพ์คำสั่งภาษาไทยหรืออังกฤษ Kiro จะสร้างโค้ดให้ทันที

วิธีใช้
  1. คลิกที่ช่อง Chat ด้านขวา
  2. ตรวจสอบว่าอยู่ในโหมด "Vibe" (ด้านบนของ Chat)
  3. พิมพ์สิ่งที่ต้องการ เช่น:
ตัวอย่าง Prompt ภาษาไทย:
สร้างไฟล์ HTML หน้าลงทะเบียน มีฟอร์มกรอก ชื่อ นามสกุล เบอร์โทร อีเมล 
ใช้ Bootstrap 5 สวยงาม มี validation ภาษาไทย
ตัวอย่าง Prompt อื่นๆ:
  • สร้าง Code.gs สำหรับอ่านข้อมูลจาก Google Sheet แล้วส่งกลับเป็น JSON
  • แก้ไขไฟล์ index.html เพิ่มตารางแสดงข้อมูล
  • อธิบายฟังก์ชัน doGet() ในไฟล์ Code.gs ให้หน่อย
  • หา bug ในไฟล์นี้ แล้วแก้ให้
โหมดการทำงาน 2 แบบ
Autopilot (ค่าเริ่มต้น) Kiro ทำงานอัตโนมัติจนเสร็จ — สร้างไฟล์, แก้ไข, รันคำสั่ง โดยไม่ถามทุกขั้นตอน
Supervised Kiro หยุดถามทุกขั้นตอน — คุณต้องกด Accept/Reject ทีละการเปลี่ยนแปลง
💡 เคล็ดลับ: ผู้เริ่มต้นแนะนำใช้ Autopilot เพื่อให้เห็นผลลัพธ์เร็ว จากนั้นค่อยเปลี่ยนเป็น Supervised เมื่อต้องการควบคุมมากขึ้น

3.4 Steering Files — กำหนดกฎให้ AI

Steering คือไฟล์กำหนดกฎ/มาตรฐานที่ AI ต้องปฏิบัติตามทุกครั้ง — เหมือน "คู่มือพนักงาน" ให้ AI

ทำไมต้องใช้ Steering?
  • ไม่ต้องพิมพ์ซ้ำทุกครั้งว่า "ใช้ Bootstrap", "เขียนภาษาไทย"
  • ทุกไฟล์ที่ AI สร้างจะเป็นมาตรฐานเดียวกัน
  • กำหนดข้อห้าม เช่น "ห้ามใช้ template literal"
วิธีสร้าง Steering File
1 สร้างโฟลเดอร์

สร้างโฟลเดอร์ .kiro/steering/ ในโปรเจกต์

2 สร้างไฟล์ .md

สร้างไฟล์ เช่น .kiro/steering/project-rules.md

# กฎการพัฒนา

## เทคโนโลยี
- ใช้ Google Apps Script (GAS) เป็น Backend
- ใช้ Google Sheets เป็นฐานข้อมูล
- ใช้ HTML5 + Bootstrap 5 เป็น Frontend
- ภาษาสื่อสาร: ไทย

## ข้อห้าม
- ห้ามใช้ template literal (backtick) ใน HTML
- ห้ามใช้ SVG ใน HTML
- ทุกลิงก์ข้ามหน้าต้องมี target="_top"

## มาตรฐาน
- ตั้งชื่อฟังก์ชันเป็น camelCase
- ตั้งชื่อ Sheet เป็นภาษาอังกฤษ UPPERCASE
- Comment เป็นภาษาไทย
💡 Steering จะถูกส่งให้ AI อัตโนมัติ ทุกครั้งที่คุณแชทกับ Kiro — ไม่ต้องพิมพ์กฎเหล่านี้ซ้ำ!

3.5 Hooks — อัตโนมัติเมื่อเกิดเหตุการณ์

Hooks คือ "กฎอัตโนมัติ" ที่สั่งให้ Kiro ทำงานบางอย่างเมื่อเกิดเหตุการณ์ เช่น:

เหตุการณ์ (Event)ตัวอย่างการใช้งาน
fileEdited — แก้ไขไฟล์รัน lint ทุกครั้งที่แก้ไขไฟล์ .ts
fileCreated — สร้างไฟล์ใหม่เพิ่ม header comment อัตโนมัติ
promptSubmit — ส่งข้อความเตือนให้ git pull ก่อนทำงาน
agentStop — ทำงานเสร็จเตือนให้ git push
userTriggered — กดปุ่มปุ่ม deploy, ปุ่ม backup
ตัวอย่าง Hook: เตือน Git Pull
{
  "name": "เตือน Git Pull",
  "version": "1.0.0",
  "when": {
    "type": "promptSubmit"
  },
  "then": {
    "type": "askAgent",
    "prompt": "เตือนให้ git pull ก่อนแก้ไขไฟล์"
  }
}
💡 วิธีสร้าง Hook: ใช้ Command Palette (Ctrl+Shift+P) → พิมพ์ "Open Kiro Hook UI" หรือบอก Kiro ใน Chat ว่า "สร้าง hook เตือนให้ push หลังทำงานเสร็จ"

3.6 MCP — เชื่อมต่อเครื่องมือภายนอก

MCP (Model Context Protocol) คือวิธีเชื่อมต่อ Kiro กับเครื่องมือภายนอก เช่น Database, API, Cloud Services

ตัวอย่างการใช้งาน MCP:
  • เชื่อมต่อกับ AWS Documentation — ค้นหาเอกสาร AWS
  • เชื่อมต่อกับ Database — อ่าน/เขียนข้อมูล
  • เชื่อมต่อกับ Google Drive — จัดการไฟล์

* MCP เป็นเนื้อหาขั้นสูง จะกลับมาเรียนรู้เพิ่มเติมในวันที่ 5
